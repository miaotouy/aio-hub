# 多模态转写中心 UI/UX 重新设计方案

## 1. 设计目标
将原有的简单转写列表重构为一个集 **即时处理**、**批量调度** 与 **精细配置** 于一体的专业转写工作台。利用 VLM（视觉语言模型）的多模态能力，为用户提供图片、音视频及 PDF 的一站式文本化方案。

## 2. 核心架构
采用 `el-tabs` 作为一级导航，分为三个主要维度：
1. **即时转写 (Workbench)**: 核心工作区，支持“所见即所得”的单文件转写。
2. **任务队列 (Queue)**: 监控后台批量处理进度，管理历史任务。
3. **全局配置 (Settings)**: 预设各模态的模型路由与 Prompt 模板。

---

## 3. 详细页面设计

### 3.1 即时转写 (Workbench) - 核心交互区
采用 **左右+下方** 的三段式布局：

#### A. 左侧：上传与预览区 (Upload & Preview)
*   **初始状态**: 显示一个巨大的 `DropZone`，支持拖拽、点击上传及剪贴板粘贴。
*   **激活状态**:
    *   **图片**: 显示可缩放预览图（参考 `SmartOcr`）。
    *   **音视频**: 显示带波形/进度条的播放器（参考 `TranscriptionDialog`）。
    *   **PDF**: 显示文档图标及基本信息。
    *   **悬浮操作**: 支持更换文件、清除当前预览。

#### B. 右侧：任务控制面板 (Control Panel)
*   **顶部 - 基础配置**:
    *   **转写模式**: 自动识别（默认）/ 图片提取 / 音频转录 / 视频分析 / PDF 解析。
    *   **语言偏好**: 中文 / 英文 / 自动。
*   **中部 - 模型配置 (折叠面板)**:
    *   **模型选择**: 使用 `LlmModelSelector` 快速切换。
    *   **高级参数**: 温度 (Temperature)、最大 Token 等。
    *   **自定义 Prompt**: 针对当前任务的临时指令追加。
*   **底部 - 动作触发**:
    *   **主按钮**: 动态显示“开始转写”、“停止/终止”、“重新生成”。
    *   **状态反馈**: 处理时的进度条或加载动画。

#### C. 下方：转写结果区 (Output Area)
*   **实时流式输出**: 模拟聊天框的打字机效果。
*   **工具栏**: 
    *   一键复制。
    *   发送到聊天。
    *   保存为资产。
    *   在编辑器中打开 (Markdown 模式)。

---

### 3.2 任务队列 (Task Queue)
*   **统计卡片**: 顶部横向排列：`正在处理`、`等待中`、`已完成`、`失败`。
*   **增强型表格**:
    *   **文件列**: 包含缩略图 + 文件名。
    *   **状态列**: 带有颜色标识的 Tag。
    *   **进度列**: 线性进度条。
    *   **操作列**: 快速查看结果 (弹窗)、重试、取消、彻底删除。

---

### 3.3 全局配置 (Settings)
*   **模态路由**: 为不同文件类型配置默认的 VLM 服务商和模型。
*   **模板仓库**: 编辑不同模态的系统 Prompt（如视频分析的框架、图片 OCR 的排版要求）。
*   **性能优化**: 并发任务数设置、FFmpeg 路径检测、自动重试机制。

---

## 4. 视觉规范
*   **质感**: 延续 `AIO Hub` 的毛玻璃效果 (`backdrop-filter`) 与深色透明背景。
*   **布局**: 响应式设计，在窄屏下自动切换为单列堆叠。
*   **反馈**: 关键节点（任务完成、出错）通过 `customMessage` 及时提醒。

---

## 5. 开发计划 (Phase 1)
1.  **组件拆分**: 建立 `components/` 目录，分离 `QuickTranscribe.vue` 和 `TranscriptionQueue.vue`。
2.  **状态同步**: 确保 `transcriptionStore` 能够实时驱动 UI 状态。
3.  **结果持久化**: 转写结果自动关联到资产的 `derived` 元数据中。
