# Agent 工具调用

Agent 工具调用让 AI 不只回答文字，还能调用 AIO Hub 内置工具、插件方法和 Skill 能力。它适合让 AI 检索知识库、分析目录、处理文件、执行检查或调用你安装的扩展服务。

## 工作方式

一次工具调用通常包含这些步骤：

1. Agent 在系统提示词中看到可用工具定义。
2. AI 回复中生成工具请求。
3. AIO Hub 解析请求并检查审批策略。
4. 用户批准或系统自动批准。
5. 工具执行，结果被写回对话。
6. AI 根据结果继续回复，或进入下一轮工具调用。

这套循环有最大轮次限制，避免模型连续调用工具停不下来。

## 开启工具调用

1. 打开 LLM Chat。
2. 进入 Agent 编辑器或聊天输入区的工具调用设置。
3. 打开工具调用总开关。
4. 选择默认策略：手动审批或自动审批。
5. 按工具启用你希望 Agent 使用的能力。

首次使用建议选择手动审批。等你确认某些工具足够安全、参数稳定后，再为它们开启自动批准。

## 审批策略

| 策略 | 适合场景 | 说明 |
| :--- | :--- | :--- |
| 手动审批 | 文件操作、网络请求、插件工具、不熟悉的 Agent | 每个请求都需要你确认 |
| 自动审批 | 只读查询、低风险统计、你信任的工具 | 命中白名单后自动执行 |
| 静默执行 | 只想拿工具结果，不希望继续生成多轮回复 | 当前轮工具执行结束后停止循环 |

审批栏中可以逐项批准、全部允许、全部拒绝，也可以切换静默模式。

## 控制工具范围

Agent 的工具调用配置可以限制：

- 是否启用工具调用。
- 最大循环轮次。
- 单次调用超时时间。
- 是否允许并行执行同一轮多个请求。
- 默认是否启用新发现的工具。
- 每个工具是否可用。
- 每个工具是否自动批准。

如果你要构建稳定的工作流，建议只打开必要工具，而不是让 Agent 看到全部能力。

## 使用 Skill

Skill 会以工具形式被披露给 Agent。模型先看到 Skill 摘要，判断需要时再激活完整说明，随后按需读取资源或执行脚本。

适合 Skill 的场景包括：

- 某类项目的固定工作流程。
- 本地脚本自动化。
- 复杂工具的使用说明和操作模板。
- 团队内部标准化处理步骤。

由于 Skill 可能执行脚本或读取文件，建议在不熟悉 Skill 内容时保留手动审批。

## 调试工具调用

如果工具调用没有按预期执行，可以这样排查：

1. 确认当前 Agent 已开启工具调用。
2. 检查目标工具是否启用。
3. 查看是否被审批栏拒绝或超时。
4. 打开工具调用测试工具，验证工具定义和参数格式。
5. 查看日志管理器，确认执行器是否记录了错误。
6. 对已有助手消息使用重新解析，让系统重新识别其中的工具请求。

## 提示词建议

让 Agent 使用工具时，描述目标、边界和风险偏好：

```text
请先检查项目目录结构，再总结文档缺口。只读取文件，不要修改任何内容。需要调用工具时请先等待我确认。
```

对于可自动执行的安全任务，可以明确授权：

```text
你可以自动调用只读分析工具，最多循环 3 轮。涉及写文件、删除文件或网络请求时必须先询问我。
```

## 常见问题

### 为什么 AI 明明说要调用工具，却没有执行？

可能是 Agent 未启用工具调用、工具不在可用列表中、模型输出格式不符合协议，或该请求被审批策略拦截。先查看审批栏和日志管理器。

### 自动批准安全吗？

自动批准取决于工具能力。只读查询通常风险较低；写文件、执行命令、访问网络或调用第三方插件时应谨慎。

### 工具调用结果会进入上下文吗？

会。执行结果会按协议追加到对话路径中，模型会基于结果继续生成。上下文管道和 Token 限制仍会影响最终注入内容。

### 可以限制 AI 最多调用几轮吗？

可以。在 Agent 的工具调用配置中设置最大迭代次数。默认限制用于防止无限循环。
