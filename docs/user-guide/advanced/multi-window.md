# 多窗口模式

多窗口模式可以把工具或局部面板从主窗口中分离出来，放到独立窗口里使用。它适合双屏工作、长时间监控、边聊边查资料，以及把输入框或监控面板固定在顺手的位置。

## 适合什么场景

- 一边在 LLM 对话中写提示词，一边把参考工具放在另一个屏幕。
- 把 VCP 连接器的消息监控面板单独悬浮，避免频繁切换标签。
- 调试组件状态同步时，将测试面板分离成独立窗口观察。
- 在长对话中，把输入区域或辅助面板放到更靠近手边的位置。

## 快速上手

### 分离整个工具

1. 在主页打开一个工具。
2. 在左侧已打开工具列表中找到该工具图标。
3. 右键工具图标，选择分离相关操作；部分工具也支持拖拽触发分离。
4. 分离窗口打开后，主窗口会保留原工具入口，方便你继续切换其他工具。

如果窗口已经打开，再次触发分离通常会聚焦已有窗口，而不是重复创建。

### 分离局部组件

部分工具提供可分离的局部组件，例如组件测试器的同步测试面板、VCP 连接器的消息监控面板，以及 LLM Chat 中适配悬浮输入体验的区域。

常见入口包括：

- 组件标题栏中的拖拽手柄。
- 面板工具栏中的分离按钮。
- 面板右键或更多菜单中的分离操作。

分离后，主窗口一般会显示占位提示或收回按钮；点击收回后，对应窗口会关闭并恢复到主窗口。

## 窗口控制

### 返回主窗口

分离窗口标题栏会提供返回主窗口的按钮。点击后，窗口关闭，主窗口中的工具或组件恢复内嵌显示。

### 聚焦已有窗口

如果你忘记某个窗口已经打开，可以再次点击分离入口。系统会优先尝试聚焦已有窗口。主窗口的标题栏和工具侧边栏也会同步已分离状态。

### 自动调整位置

如果启用了窗口位置自动调整，AIO Hub 会定期检查分离窗口是否仍在可见屏幕范围内。外接显示器断开、分辨率变化后，这个功能能减少窗口跑到屏幕外的情况。

入口：设置 -> 通用与启动项，查找窗口位置相关选项。

## 状态同步

多窗口不是简单复制界面。主窗口、分离工具窗口、分离组件窗口之间会通过同步总线交换状态：

- 主窗口和分离工具窗口可以作为状态源。
- 分离组件窗口会向父窗口请求初始状态。
- 组件里的操作会通过远程 Action 回到父窗口执行。
- 设置、主题和窗口状态会尽量保持一致。

因此，在分离输入框或监控面板中完成的操作，会同步影响原工具的实际状态。

## 使用建议

- 把高频输入区域放在主屏，把监控或参考工具放在副屏。
- 长时间运行的监控窗口不要开太多，避免桌面杂乱。
- 如果发现窗口内容不同步，先收回再重新分离。
- 如果分离窗口无法找回，回到设置中开启窗口位置自动调整，或重启应用清理旧状态。

## 常见问题

### 为什么有些工具不能分离局部面板？

只有工具显式注册了可分离组件，局部面板才会出现分离入口。没有注册的工具仍可尝试分离整个工具窗口。

### 关闭分离窗口会丢数据吗？

正常关闭会触发重新附着流程，状态会回到主窗口。尚未保存的编辑内容仍建议先保存或发送后再关闭。

### 分离窗口中的设置按钮会打开哪里？

在分离窗口中打开设置时，系统会尽量聚焦主窗口并跳转到对应设置页，避免在多个窗口里同时编辑全局设置。

### 可以把同一个组件分离多次吗？

通常不建议也不会重复创建。系统会根据组件 ID 识别已有窗口，重复操作会优先聚焦现有窗口。
