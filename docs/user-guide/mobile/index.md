# 移动端概览

AIO Hub Mobile 是面向手机和平板的移动端实现，基于 Tauri v2 和 Vue 3 构建。移动端不会完整复制桌面端的所有工具，而是优先提供适合触屏使用的核心能力。

## 当前可用模块

| 模块 | 用途 |
| :--- | :--- |
| LLM 对话 | 新建会话、继续历史会话、发送消息、查看分支 |
| LLM 服务 | 管理渠道、模型、Key 和自定义端点 |
| 日志管理 | 查看、搜索、过滤、导出运行日志 |
| UI 测试 | 用于开发验证移动端组件效果 |

移动端功能仍在持续对齐桌面端。遇到桌面端教程中没有出现在移动端的按钮或工具时，以移动端实际界面为准。

## 首次使用

1. 打开移动端应用。
2. 在首页工具网格中进入「LLM 服务」。
3. 新增一个服务渠道，填写 API Key 和模型。
4. 回到首页，进入「AI 对话」。
5. 点击「开启新对话」，发送第一条消息。

如果你已经熟悉桌面端，可以把移动端理解为更轻量的随身对话面板：配置服务、发起会话、查看日志是目前最稳定的主线。

## 界面结构

移动端采用底部导航和触屏卡片布局：

- 首页：展示已注册工具。
- 设置：主题、外观和应用级设置。
- 工具页：每个工具有独立路由和顶部栏。
- 弹层：模型选择、渠道编辑等复杂表单使用底部弹层或全屏编辑体验。

移动端会处理安全区和键盘避让。输入时底部操作栏会贴近键盘，减少被遮挡的情况。

## 与桌面端的差异

- 移动端更强调单任务流程，不提供桌面端的多窗口分离。
- 部分工具尚未移植。
- UI 骨架由移动端自研 Vue 组件和 AIO Hub 主题 token 主导；Varlet 仅作为底层组件库按需使用。
- 移动端源码采用独立许可，发布和二次分发限制与桌面端不同。
- 路由和语言包单独维护，工具名称可能与桌面端略有差异。

## 使用建议

- 先在「LLM 服务」配置好至少一个可用渠道。
- 长文本编辑建议分段发送，避免移动键盘操作压力过大。
- 出现异常时，优先查看「日志管理」。
- 如果模型列表为空，可以手动添加模型，或通过渠道编辑器获取模型。
- 移动网络不稳定时，适当降低并发或切换更稳定的服务端点。

## 常见问题

### 移动端支持全部桌面工具吗？

暂不支持。移动端当前优先覆盖 AI 对话、LLM 服务配置和日志查看等核心流程。

### 移动端和桌面端数据会自动同步吗？

当前教程不假设自动同步。请以移动端本地配置为准；跨端同步能力如果后续加入，会在对应页面说明。

### 为什么某些教程截图和移动端不一样？

大多数工具教程以桌面端为主。移动端使用单独布局，请优先参考移动端章节。

### 移动端可以商用或二次分发吗？

移动端目录采用私有许可证，未获授权不得商用、二次分发或套壳发布。
