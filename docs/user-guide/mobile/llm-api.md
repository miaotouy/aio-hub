# API 配置（移动端）

移动端的「LLM 服务」用于管理 AI 服务渠道、模型和 API Key。它是移动端 LLM 对话能否正常发送请求的前置配置。

## 打开配置页

1. 在移动端首页点击「LLM 服务」。
2. 如果还没有渠道，页面会显示空状态。
3. 点击右上角加号。
4. 选择一个预设服务商，或创建自定义渠道。

## 新建渠道

选择预设后，系统会带入服务商名称、接口类型、默认 Base URL 和部分默认模型。你需要补充：

- API Key。
- 需要使用的模型。
- 是否启用该渠道。
- 如有需要，配置自定义请求头和端点。

如果你的服务商兼容 OpenAI 格式，可以选择 OpenAI 兼容类型，并填写服务商提供的 Base URL。

## 编辑渠道

点击渠道卡片进入编辑弹层。常用操作包括：

- 修改渠道名称。
- 启用或禁用渠道。
- 增删 API Key。
- 管理模型列表。
- 设置图标。
- 配置自定义 Header。
- 配置自定义 Endpoint。

保存后，LLM 对话会读取已启用渠道和模型。

## 管理模型

模型列表决定聊天页可选的模型名称。你可以：

- 使用预设模型。
- 手动新增模型名称。
- 通过模型获取功能从服务商接口拉取。
- 编辑模型能力元数据。

如果服务商不提供模型列表接口，手动添加模型即可。

## Key 管理

每个渠道可以保存一个或多个 API Key。使用建议：

- 不要把同一个 Key 写到多个不需要的渠道里。
- 失效 Key 及时删除。
- 如果服务商限制频率，可以准备多个 Key 并按渠道策略管理。
- 不要把真实 Key 填入截图或公开 issue。

## 自定义端点

自定义端点用于适配不同服务商的路径差异，例如聊天、模型列表或媒体能力接口。移动端会尽量复用桌面端 LLM API 的核心请求构建逻辑。

不确定时，先保持默认端点，只改 Base URL 和 API Key。

## 排查请求失败

1. 确认渠道已启用。
2. 确认 API Key 不为空。
3. 确认模型名与服务商文档一致。
4. 确认 Base URL 没有重复路径，例如不要把 `/v1` 写两遍。
5. 打开日志管理器，搜索渠道名称或错误状态码。

## 常见问题

### 为什么添加渠道后聊天页仍然没有模型？

检查渠道是否启用，以及模型列表是否为空。部分预设不会自动带入所有模型，需要手动添加。

### Base URL 应该填到哪里？

填服务商 API 根地址。OpenAI 兼容服务通常类似 `https://example.com/v1`。如果请求日志显示路径重复，删掉多余的一段。

### 支持多个服务商吗？

支持。你可以创建多个渠道，并在聊天设置或模型选择器中切换。

### 移动端配置和桌面端配置是否共用？

当前按移动端本地配置理解。跨端迁移或同步请以未来导入导出功能说明为准。
