# 日志管理器（移动端）

日志管理器用于查看移动端运行期间产生的日志。遇到请求失败、页面异常、配置没有生效时，它是最直接的排查入口。

## 打开日志管理器

1. 在移动端首页点击「日志管理」。
2. 页面顶部是搜索和级别筛选。
3. 中间区域显示日志列表。
4. 底部提供清空、导出、复制全部等操作。

## 搜索日志

在顶部搜索框输入关键词，可以按日志内容或模块名称过滤。例如：

- `llm-api`：查看 LLM 服务相关日志。
- `llm-chat`：查看聊天相关日志。
- `ERROR`：配合级别筛选查找错误。
- 模型名或渠道名：定位某次请求。

搜索会实时过滤当前内存中的日志。

## 按级别筛选

日志级别包括：

| 级别 | 含义 |
| :--- | :--- |
| DEBUG | 调试细节，适合开发和深入排查 |
| INFO | 正常流程记录 |
| WARN | 可恢复的问题或潜在风险 |
| ERROR | 已失败的操作或异常 |

日常排查可以先选 `ERROR`，没有结果再切换到 `WARN` 或全部级别。

## 自动滚动

开启自动滚动后，新日志出现时列表会滚到底部。它适合观察请求发送、模型响应或页面加载流程。

如果你正在阅读上方的旧日志，可以先关闭自动滚动，避免列表跳走。

## 导出与复制

底部操作栏提供：

- 清空日志：清除当前内存日志。
- 导出日志：下载文本文件，适合提交 issue 或长期保存。
- 复制全部：把当前日志文本复制到剪贴板，适合快速粘贴给开发者。

提交问题时，请先检查日志中是否包含 API Key、私人路径、聊天内容等敏感信息，必要时手动删掉。

## 常见排查流程

### LLM 请求失败

1. 清空日志。
2. 回到 LLM 对话重新发送一条短消息。
3. 打开日志管理器。
4. 筛选 `ERROR`。
5. 查看状态码、渠道名、模型名和错误消息。

### 配置保存后没生效

1. 搜索配置模块名称。
2. 查看是否出现保存成功日志。
3. 返回对应工具重新进入，触发配置重新加载。
4. 如果仍异常，导出日志并反馈。

### 页面卡住或空白

1. 切到日志管理器。
2. 查看最近的 `ERROR` 和 `WARN`。
3. 记录触发路径：哪个页面、点击了哪个按钮、是否刚刚切换网络。

## 常见问题

### 清空日志会删除历史文件吗？

移动端日志管理器主要管理当前运行期间的内存日志。清空后当前列表会变空，已经导出的文件不会被删除。

### 为什么没有任何日志？

可能是刚启动还没有触发操作，或当前筛选条件太严格。切换到全部级别，并清空搜索关键词再看。

### 导出的日志在哪里？

导出会走移动端 WebView 的下载能力，具体保存位置取决于系统和运行环境。如果不确定，使用「复制全部」通常更直接。

### 日志里能看到 API Key 吗？

正常情况下不应主动记录完整 Key，但不同错误返回可能包含敏感上下文。分享日志前请自行检查。
