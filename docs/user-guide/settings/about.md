## 检查更新

### 手动检查

点击版本号旁边的「检查更新」按钮，系统会：

1. 向 GitHub Releases API 发起请求，获取最新版本信息
2. 对比当前版本和最新版本的语义化版本号
3. 如果存在新版本 → 弹出更新弹窗
4. 如果已是最新版 → 提示"当前已是最新版本"

### 强制检测模式

按住 **Alt 键** 的同时点击「检查更新」按钮，会**强制显示更新弹窗**（即使没有新版本）。这在以下场景中很有用：

- 想要查看最新的 Release Notes
- 确认更新检测功能正常工作

### 更新弹窗

弹窗包含：

| 内容         | 说明                                                                  |
| :----------- | :-------------------------------------------------------------------- |
| **标题**     | "发现新版本 vX.Y.Z"                                                   |
| **更新日志** | 使用 `RichTextRenderer` 渲染 GitHub 的 Release Notes（支持 Markdown） |
| **按钮**     | 「暂不更新」/「前往下载」→ 跳转到 GitHub Releases 页面                |

> 更新弹窗使用项目的 **BaseDialog** 组件，尺寸为 70vw × 80vh，确保更新日志有足够的阅读空间。

### 更新方式

AIO Hub 目前通过**下载最新安装包覆盖安装**的方式进行更新：

1. 在更新弹窗中点击「前往下载」
2. 跳转到 GitHub Releases 页面下载最新版本
3. 安装新版本，配置和数据不会丢失

## 支持项目

AIO Hub 是一个**完全免费的开源项目**，作者几乎全职投入开发。如果你喜欢这个项目，可以通过以下方式支持：

### 支持带来的好处

- 🚀 持续添加新功能和工具
- 🐛 及时修复问题和优化性能
- 📚 完善文档和使用教程
- 💡 探索更多创新想法

### 支持方式

| 方式                   | 说明                                               |
| :--------------------- | :------------------------------------------------- |
| **⭐ 给项目点个 Star** | 在 GitHub 上点亮 Star，这是对作者最大的鼓励        |

- GitHub Star → `https://github.com/miaotouy/aio-hub`

## 常见问题

### Q：如何获取应用的最新版本？

关注 GitHub Releases 页面即可。也可以在「关于」页面中点击「检查更新」按钮手动检查。

### Q：更新会丢失我的配置吗？

**不会。** 覆盖安装新版本后，所有配置和数据会保留。但为了保险起见，建议在重大更新前使用「通用设置 → 导出配置」功能备份配置。

### Q：如何查看许可证详情？

项目使用 MIT License，可以在项目根目录的 `LICENSE` 文件中查看完整内容。

### Q：我想参与项目开发或提建议？

欢迎你在 GitHub 上提交 Issue 或 Pull Request！仓库地址在「关于」页面中有展示。
