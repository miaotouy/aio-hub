# 通用与启动项

> 通用设置控制 AIO Hub 的核心运行行为——从界面语言、窗口行为到网络代理和配置备份。启动项管理则让你控制各工具模块的自动启动策略。

---

## 通用设置

1. 点击标题栏右侧的 ⚙️ 图标进入设置。
2. 在设置左侧导航栏中点击「通用设置」选项卡。

### 配置管理

设置面板顶部提供三个核心操作按钮：

| 按钮             | 说明                                                                                                                   |
| :--------------- | :--------------------------------------------------------------------------------------------------------------------- |
| **打开配置目录** | 快速跳转到配置文件所在目录                                                                                             |
| **导出配置**     | 将所有模块的配置打包为 ZIP 压缩包并保存到指定位置，导出的 ZIP 文件名会带上日期（如 `AIO-Tools-Backup-2025-10-01.zip`） |
| **导入配置**     | 从 ZIP 文件恢复配置，可选择合并或覆盖模式                                                                              |

#### 导入模式说明

选择导入配置时，系统会弹出提示让你选择导入模式：

| 模式         | 行为                                                                           |
| :----------- | :----------------------------------------------------------------------------- |
| **合并导入** | 保留现有配置，仅更新导入文件中存在的项。适合从旧版本迁移后要保留新增配置的情况 |
| **覆盖导入** | 完全替换为导入文件中的配置。适合完全恢复或设置迁移                             |

> 建议在重大版本更新前**先导出备份配置**，以防万一。

### 窗口与托盘

| 配置项               | 说明                                                   | 注意事项                     |
| :------------------- | :----------------------------------------------------- | :--------------------------- |
| **显示托盘图标**     | 是否在系统托盘显示应用图标                             | 关闭后，关闭到托盘功能不可用 |
| **关闭到托盘**       | 点击关闭按钮时最小化到托盘而非退出                     | 需先启用「显示托盘图标」     |
| **窗口位置记忆**     | 点击「清除窗口状态」按钮，重置所有窗口的位置和大小记忆 | 下次打开窗口时恢复默认位置   |
| **自动调整窗口位置** | 工具窗口移出屏幕时自动拉回可见区域                     | 多屏切换时很有用             |

## 启动项管理

启动项管理位于通用设置面板中，是一个独立的模块，用于控制各工具模块在应用启动时的自动初始化行为。

### 界面展示

列表显示所有实现了自动启动能力的工具模块，每项包含：

| 信息             | 说明                 |
| :--------------- | :------------------- |
| **名称**         | 工具模块名称         |
| **开关**         | 启用/禁用该启动项    |
| **上次启动结果** | 成功 ✓ / 失败 ✗      |
| **耗时**         | 上次启动的耗时       |
| **错误信息**     | 启动失败时的错误详情 |

### 熔断保护机制

启动项管理内置**熔断保护**——如果某个工具连续多次启动失败，系统会自动禁用该启动项，防止反复失败影响应用整体启动速度。

- 被熔断的工具会保留 `consecutiveFailures`（连续失败次数）和 `lastError`（最后错误信息）
- 你可以手动重新启用被熔断的启动项

### 使用建议

- 对于**必需**的工具（如资产管理），建议保持开启
- 对于**偶尔使用**的工具，可以关闭自动启动，使用时再手动打开
- 如果某个工具总是启动失败，检查其配置是否正确，或查看错误信息定位问题

## 常见问题

### Q：修改通用设置后需要重启应用吗？

**不需要。** 大部分设置项修改后即时生效。少数设置（如侧边栏模式切换）可能需要重新加载界面。

### Q：配置导出的 ZIP 文件包含哪些内容？

包含所有设置模块的配置文件（通用、外观、AI 服务、日志、资产管理等），但不包含用户数据文件（如聊天记录、用户资产文件等）。

### Q：为什么「关闭到托盘」选项是灰色的？

因为「显示托盘图标」处于关闭状态。先打开「显示托盘图标」，再配置关闭到托盘行为。

### Q：启动项熔断后如何恢复？

在启动项列表中找到被禁用的工具，手动开启它的开关即可。建议先检查配置解决失败原因再启用。
