# 设置概览

> **设置系统**是 AIO Hub 的配置中枢，所有个性化定制和行为控制都集中在这里。本篇将带你了解设置界面的整体布局和各选项卡的作用。

---

## 界面布局

点击应用顶部标题栏右侧的**设置图标** ⚙️，即可进入设置页面。

设置界面在桌面端采用**左侧导航 + 右侧内容**的经典布局示例：

```
┌─────────────────────────────────────────┐
│  ┌────────┐  ┌────────────────────────┐ │
│  │ 通用   │  │                        │ │
│  │ 外观   │  │      设置内容面板      │ │
│  │ AI服务 │  │                        │ │
│  │ 日志   │  │                        │ │
│  │ 工具   │  │                        │ │
│  │ 关于   │  │                        │ │
│  └────────┘  └────────────────────────┘ │
└─────────────────────────────────────────┘
```

**左侧导航栏**按功能分类排列所有设置选项卡，点击即可切换。**右侧内容面板**显示当前选项卡的具体配置项。

## 选项卡一览

### ⚡ 通用与启动项

涵盖应用的**核心运行时行为**，包括：

- **配置管理**：打开配置目录、导出/导入配置（ZIP 打包，支持合并/覆盖两种模式）
- **窗口与托盘**：显示托盘图标、关闭到托盘行为
- **基础主题**：跟随系统/浅色/深色模式切换
- **侧边栏模式**：默认/抽屉/下拉菜单三种布局
- **网络代理**：跟随系统/不使用/自定义代理（支持自定义代理 URL）
- **时区设置**：跟随系统或手动选择（支持全球时区列表）
- **下载管理**：默认下载路径、总是询问保存位置、下载完成通知等
- **启动项管理**：各工具的自动启动开关，内置熔断保护（连续失败自动禁用）

> 详细说明见 [通用与启动项](./general-startup.md)。

### 🎨 外观与个性化

应用的**视觉定制中心**，涵盖主题色、壁纸、界面质感和样式覆盖：

**主题色配置**：

- 五种颜色类型（主题/成功/警告/危险/信息）的预设与自定义
- 壁纸主题色自动提取联动

**壁纸管理**：

- 静态壁纸 / 轮播模式（支持间隔、随机/顺序播放）
- 内置壁纸（预设网格选择）/ 自定义（文件选择、目录选择）
- 四种填充模式（覆盖/包含/拉伸/平铺），拼贴模式附带缩放、旋转、翻转

**界面质感**：

- UI 特效全局开关（透明度与模糊）
- 基础不透明度 / 分离窗口独立不透明度
- 背景色叠加（支持 16 种 CSS 混合模式、屏幕吸色、壁纸自动取色）
- 主题色自动提取（柔和/鲜艳/亮部/暗部四种策略）
- UI 模糊（毛玻璃）开关 + 强度控制
- 边线不透明度与宽度、代码块背景不透明度

**窗口特效（实验性）**：

- 模糊 (Blur) / 亚克力 (Acrylic) / 云母 (Mica) 原生窗口效果
- 窗口背景独立不透明度、窗口阴影开关

**CSS 样式覆盖**：

- 通过编写自定义 CSS 深度定制应用界面，支持预设管理和实时预览

> 详细说明见 [外观与壁纸](./appearance.md) 和 [CSS 样式覆盖](./custom-css.md)。

### 🧩 工具模块

**工具显示与排列管理面板**：

- **可见性控制**：工具卡片显示/隐藏开关，支持全选/全不选批量操作
- **排序管理**：拖拽 ↕ 排序手柄调整主页显示顺序，同步更新侧边栏
- **重置顺序**：一键恢复默认工具排列

> 详细说明见 [工具管理](./tool-management.md)。

### 🔑 AI 服务配置

**LLM 服务渠道管理器**，涵盖以下独立配置模块：

- **[LLM AI 服务配置](./llm-service.md)**：管理 API 提供商的渠道信息，支持多渠道 CRUD、拖拽排序、多 API Key 负载均衡、模型管理、连接测试。
- **[AI 模型元数据配置](./model-metadata.md)**：通过规则匹配为模型分配图标和分组，支持四种匹配类型（Provider/Model/Prefix/Group）和正则表达式。
- **[云端 OCR 服务](./ocr-service.md)**：管理 OCR 服务渠道，支持标准 API 和自定义 API 请求配置。
- **[用户档案管理](./user-profiles.md)**：管理 LLM 聊天的用户档案，包含头像上传、富文本样式行为配置。

### 📋 日志与资产管理

**日志系统**和**资产存储配置**的管理面板：

**日志设置**：

- 日志级别（DEBUG/INFO/WARN/ERROR，实时生效）
- 文件日志与控制台日志独立开关
- 缓冲区大小配置（500-50000 条）、文件分割阈值（1MB-20MB）
- 实时日志统计面板（各类别日志条数）
- 操作入口：打开日志目录、导出日志、清空缓冲区、生成测试日志

**资产管理**：

- 资产存储路径配置（显示默认路径 + 自定义路径选择）
- 路径缓存自动刷新，重置一键恢复

> 详细说明见 [日志与资产管理](./logs-assets.md)。

### ℹ️ 关于与许可

**应用信息与支持页面**：

- 名称与版本信息（来自 Tauri API）
- 检查更新（从 GitHub Releases API 获取，支持 Semantic Version 对比）
- 项目链接（开发者、仓库地址、许可证）
- 支持项目（GitHub Star、爱发电赞助）
- 版权信息

> 详细说明见 [关于与许可](./about.md)。

## 配置持久化说明

AIO Hub 的所有设置都存储在本地文件系统中，**不会上传到任何远程服务器**。

### 存储位置

配置文件位于应用的**数据目录**下，你可以通过设置中的「打开配置目录」按钮快速访问。

### 备份与迁移

通过通用设置中的**配置导出/导入**功能，你可以：

1. **导出配置**：将所有模块的配置打包为 ZIP 压缩包
2. **导入配置**：支持两种模式——
   - **合并导入**：保留现有配置，仅更新导入文件中存在的项
   - **覆盖导入**：完全替换为导入文件中的配置

> 建议在重大版本更新前备份配置，以便随时恢复。

### 实时生效

大多数设置项（如主题切换、日志级别、代理模式等）修改后**即时生效**，无需重启应用。部分设置（如资产存储路径）仅影响后续操作，已有数据不会自动迁移。
