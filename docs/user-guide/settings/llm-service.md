# AI 服务配置

> AIO Hub 的所有 AI 功能（LLM 对话、智能 OCR、知识库检索等）都依赖这里配置的服务渠道。本篇将带你完成从添加 API Key 到模型管理的完整配置流程。

---

## 入口

1. 点击标题栏右侧的 ⚙️ 图标进入设置。
2. 在左侧导航栏中找到对应的 AI 相关选项卡：

| 选项卡                | 功能                               |
| :-------------------- | :--------------------------------- |
| **LLM AI 服务配置**   | 管理 LLM API 提供商渠道            |
| **AI 模型元数据配置** | 为模型分配图标和分组标签的规则引擎 |
| **云端 OCR 服务**     | 管理云端 OCR 服务渠道              |
| **用户档案管理**      | 管理 LLM 聊天中的用户档案          |

下面逐一说明每个子面板的用法。

---

## LLM 服务配置

这是使用 AIO Hub LLM 对话功能前的**核心前提配置**。

### 界面布局

```
┌────────────────────┬──────────────────────────────────────────┐
│   渠道列表（侧栏） │    渠道编辑区（右侧主面板）              │
│                    │                                          │
│  ┌──────────────┐  │  ┌────────────────────────────────────┐  │
│  │ OpenAI       │  │  │  # 基本信息                        │  │
│  │ DeepSeek     │  │  │  · 渠道名称、API 类型、图标        │  │
│  │ Claude       │  │  │  · API 地址                        │  │
│  │ + 新建渠道   │  │  ├────────────────────────────────────┤  │
│  └──────────────┘  │  │  # 网络设置（折叠）                │  │
│                    │  │  · 代理方案、证书校验、请求头      │  │
│                    │  ├────────────────────────────────────┤  │
│                    │  │  # 多密钥管理                      │  │
│                    │  │  · 逗号分隔/弹窗管理               │  │

└────────────────────┴──────────────────────────────────────────┘
```

### 步骤一：创建渠道

1. 点击渠道列表顶部的「新建渠道」按钮
2. 选择创建方式：
   - **预设模板**：从常用服务商（如 OpenAI、DeepSeek、Claude 等）的预设模板中选择，大部分配置自动填充
   - **空白**：从零开始手动配置
3. 渠道创建后，在右侧编辑区完善配置

### 步骤二：配置基本信息

| 字段             | 说明                                       |
| :--------------- | :----------------------------------------- |
| **渠道名称**     | 自定义名称，方便识别                       |
| **API 格式类型** | 选择 API 兼容的格式类型（如 OpenAI-style） |
| **供应商图标**   | 从预设图标选择器中选取服务商标识           |
| **API 地址**     | 服务的 API 端点，支持快捷链接和端点预览    |

### 步骤三：配置网络（可选）

在折叠的「网络设置」面板中，你可以：

| 配置项            | 说明                        |
| :---------------- | :-------------------------- |
| **网络方案**      | 自动 / 后端代理 / 原生请求  |
| **放宽证书校验**  | 用于自签名证书或内网服务    |
| **强制 HTTP/1.1** | 针对某些老旧 API 的兼容选项 |
| **自定义请求头**  | 添加自定义的 HTTP 请求头    |
| **高级端点**      | 覆盖默认的 API 路径         |

> 大多数情况下保持默认（自动）即可。

### 步骤四：管理 API Key

AIO Hub 支持**多密钥管理**：

1. 在密钥输入框中输入 API Key
2. 支持**逗号分隔**输入多个 Key，实现负载均衡
3. 点击「管理密钥」按钮，在弹窗中：
   - 查看所有密钥的启用/禁用状态
   - 单独测试每个密钥的连通性
   - 启用/禁用单个密钥

> 所有 API Key 都存储在本地，不会上传到任何服务器。

### 步骤五：配置模型

模型管理是渠道配置中最重要的一环：

| 操作                | 说明                                    |
| :------------------ | :-------------------------------------- |
| **添加模型**        | 手动输入模型 ID、名称、上下文长度等信息 |
| **从 API 获取**     | 自动拉取服务商支持的模型列表            |
| **编辑模型**        | 修改模型参数                            |
| **删除 / 批量删除** | 移除模型                                |
| **分组展开**        | 按组管理多个模型                        |
| **模型连接测试**    | 快速测试单个模型的连通性                |

### 连接测试

所有配置完成后，你可以：

- **一键测试**：测试整个渠道的连接状态
- **单模型测试**：针对特定模型测试
- **单密钥测试**：针对特定 API Key 测试（在多密钥场景下很有用）

## 模型元数据规则引擎

这个子面板用于控制**模型在 UI 中的图标和分组显示**。系统通过规则匹配自动为模型分配图标。

### 规则匹配类型

| 类型         | 匹配方式           | 示例     |
| :----------- | :----------------- | :------- |
| **Provider** | 匹配服务提供商名称 | `openai` |
| **Model**    | 精确匹配模型 ID    | `gpt-4o` |
| **Prefix**   | 匹配模型 ID 前缀   | `gpt-`   |
| **Group**    | 匹配分组名称       | `chat`   |

支持**正则表达式**。规则按优先级排序，匹配成功后立即生效。

### 操作界面

- **网格/列表** 双视图切换
- **搜索过滤**：按关键字、启用状态搜索
- **排序**：按优先级、类型、名称、创建时间
- **分页**：管理大量规则时保持界面清晰

### 常用操作

| 操作                 | 说明                                   |
| :------------------- | :------------------------------------- |
| **添加规则**         | 选择匹配类型，输入匹配值，选择预设图标 |
| **编辑/删除**        | 修改已有规则                           |
| **启用/禁用**        | 临时关闭某条规则而不删除               |
| **重置为默认**       | 恢复内置规则                           |
| **合并最新内置规则** | 在应用更新后，合并新版内置规则         |
| **导入/导出 JSON**   | 批量管理规则集                         |

### 测试模式

在测试模式下，输入**模型 ID + Provider**，系统会实时展示：

- 匹配状态和命中的规则
- 匹配到的图标预览
- 候选规则列表

这在你新增模型后想确认图标是否正确时非常有用。

## 云端 OCR 服务配置

如果你需要使用智能 OCR 工具的云端识别引擎，需要先在这里配置 OCR 服务渠道。

### 标准服务商

| 字段                 | 说明                                |
| :------------------- | :---------------------------------- |
| **服务名称**         | 自定义名称                          |
| **服务类型**         | 选择预设的 OCR 服务商（如百度云等） |
| **API 端点**         | 服务地址                            |
| **API Key / Secret** | 认证凭据                            |

### 自定义 API 支持

如果标准服务商不满足需求，你可以配置**自定义 API**：

| 配置项             | 说明                            |
| :----------------- | :------------------------------ |
| **自定义 API URL** | 请求目标地址                    |
| **HTTP 方法**      | GET / POST 等                   |
| **请求头**         | 自定义请求头                    |
| **请求体模板**     | 支持 `{{imageBase64}}` 变量占位 |
| **结果提取路径**   | 从响应 JSON 中提取识别结果      |

### 并发与限流

| 配置项       | 说明                                     |
| :----------- | :--------------------------------------- |
| **并发数**   | 1-10，控制图片块并行处理数               |
| **请求延迟** | 0-5000ms，每次请求间的延时，避免触发限流 |

所有配置变更会在 1 秒防抖后**自动保存**。

## 用户档案管理

用户档案用于 LLM 聊天场景，为 AI 对话提供用户侧的描述信息。

### 档案列表

侧边栏按最近使用排序，显示：

- 头像
- 名称
- 描述（截断）
- 创建日期

### 编辑档案

点击档案进入编辑模式，支持：

- **头像上传/清除**
- **富文本样式行为配置**：定义用户在对话中的发言样式
- **正则预置管理**：管理用户消息的预处理规则

### 其他操作

| 操作         | 说明                                 |
| :----------- | :----------------------------------- |
| **新建档案** | 弹窗输入名称、显示名、简介、头像 URL |
| **删除档案** | 确认后删除到回收站                   |
| **打开目录** | 直接打开档案配置文件所在目录         |

所有表单变更会在 1 秒防抖后自动保存。

## 常见问题

### Q：配置完成后在哪里使用 AI 功能？

在 **LLM 智能对话** 工具中，选择你配置好的渠道和模型即可开始对话。

### Q：多 API Key 的负载均衡策略是什么？

系统会按顺序轮询使用可用的 API Key。如果某个 Key 请求失败，自动切换到下一个 Key。

### Q：模型元数据规则的作用是什么？

它控制模型在编辑器和对话框中的显示图标和分组，对于区分同一提供商的不同模型家族很有用（如分清 GPT-4o 和 o1 系列）。

### Q：OCR 服务配置和 LLM 服务配置分别用于哪里？

- **LLM 服务配置**：用于 LLM 智能对话、知识库 Agent 等依赖语言模型的功能
- **OCR 服务配置**：用于智能 OCR 工具的云端识别引擎
