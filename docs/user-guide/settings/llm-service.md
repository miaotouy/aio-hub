# LLM AI 服务配置

> 这是使用 AIO Hub LLM 对话功能前的**核心前提配置**。管理 LLM API 提供商渠道，支持多渠道管理和快速连接测试。

---

## 入口

1. 点击标题栏右侧的 ⚙️ 图标进入设置。
2. 在左侧导航栏中选择 **LLM AI 服务配置**。

## 步骤一：创建渠道

点击渠道列表顶部的「新建渠道」按钮，选择创建方式：

- **渠道预设**：从常用服务商（如 OpenAI、DeepSeek、Claude 等）的预设模板中选择，API 地址、快捷链接等配置将自动填充；也可以从空白配置开始。
- **粘贴导入**：粘贴 cURL、环境变量、JSON 或 TOML，也可以一次选择多个配置文件。自动检测支持 Claude Code、Gemini CLI、Codex、Grok CLI 和 OpenCode 常见配置。

导入前会列出识别到的候选渠道、API 格式、Base URL、模型和 Key 配置状态。多 provider 配置可勾选多个候选批量创建；缺少 Base URL 的候选不能导入，缺少 Key 的候选可以创建后再补充。

Codex 的 `config.toml` 与 `auth.json` 可以在同一次文件选择中导入，也可以前后粘贴到同一个输入框。粘贴内容可直接拼接，或使用 `config.toml` / `auth.json` 标题分段。只有配置和凭据均唯一时才会自动配对，存在歧义时会保留候选并提示人工核对。

编辑现有渠道时，可使用 API 地址右侧的导入按钮。导入结果只覆盖明确提供的 Key、模型、自定义请求头和端点，不会默认清空现有字段。

---

## 步骤二：配置基本信息

| 字段             | 说明                                                                          |
| :--------------- | :---------------------------------------------------------------------------- |
| **渠道名称**     | 自定义名称，方便在对话工具中识别。                                            |
| **API 格式类型** | 选择兼容的格式（如 OpenAI-style, Google Gemini, Anthropic 等）。              |
| **供应商图标**   | 自定义图标路径或从预设图标库中选取。                                          |
| **API 地址**     | 服务的 API 端点。点击「重置」可恢复该类型的默认地址。下方会实时显示端点预览。 |

---

## 步骤三：填写 API Key

在密钥输入框中填入你的 API Key。如果有多个 Key，使用英文逗号 `,` 分隔即可。

> 💡 多密钥的高级管理（健康度监控、自动熔断、负载均衡）请参阅 [高级配置 — 多密钥管理](./llm-service-advanced.md#多密钥管理与负载均衡)。

---

## 步骤四：添加模型

点击「添加模型」或使用「获取模型列表」从 API 自动拉取可用模型。

每个模型的基本设置：

- **模型 ID**：API 调用时的真实 ID。
- **应用预设**：输入 ID 后点击「应用预设」，系统将自动匹配并填充名称、分组、图标和模型能力。

> 💡 模型的能力开关、Token 限制、价格和自定义参数等高级配置请参阅 [高级配置 — 深度模型配置](./llm-service-advanced.md#深度模型配置)。

---

## 连接测试

点击 API Key 旁的「测试连接」按钮，系统将使用配置的第一个模型进行快速握手，验证渠道是否可用。

---

## 常见问题

### Q：配置完成后在哪里使用 AI 功能？

在 **LLM 智能对话** 工具中，选择你配置好的渠道和模型即可开始对话。

### Q：网络连接失败怎么办？

1. 确认 API 地址填写正确（注意末尾不要多加 `/`）。
2. 如果需要代理或特殊网络设置，请参阅 [高级配置 — 网络设置](./llm-service-advanced.md#网络设置)。

### Q：支持哪些 API 格式？

目前支持 OpenAI-style、Google Gemini、Anthropic 等主流格式，覆盖绝大多数 LLM 服务商。
