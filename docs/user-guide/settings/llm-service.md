# LLM AI 服务配置

> 这是使用 AIO Hub LLM 对话功能前的**核心前提配置**。管理 LLM API 提供商渠道，支持多渠道管理、多密钥负载均衡及深度模型配置。

---

## 入口

1. 点击标题栏右侧的 ⚙️ 图标进入设置。
2. 在左侧导航栏中选择 **LLM AI 服务配置**。

## 步骤一：创建渠道

点击渠道列表顶部的「新建渠道」按钮，选择创建方式：

- **预设模板**：从常用服务商（如 OpenAI、DeepSeek、Claude 等）的预设模板中选择，API 地址、快捷链接等配置将自动填充。
- **空白**：从零开始手动配置。

---

## 步骤二：配置基本信息与网络

### 1. 核心字段

| 字段             | 说明                                                                          |
| :--------------- | :---------------------------------------------------------------------------- |
| **渠道名称**     | 自定义名称，方便在对话工具中识别。                                            |
| **API 格式类型** | 选择兼容的格式（如 OpenAI-style, Google Gemini, Anthropic 等）。              |
| **供应商图标**   | 自定义图标路径或从预设图标库中选取。                                          |
| **API 地址**     | 服务的 API 端点。点击「重置」可恢复该类型的默认地址。下方会实时显示端点预览。 |

### 2. 网络设置（折叠面板）

针对复杂的网络环境，AIO Hub 提供了深度微调选项：

- **网络方案**：
  - **自动选择/后端代理**：默认推荐。使用 Rust 后端代理请求，可绕过浏览器 CORS 限制，支持底层网络微调。
  - **原生请求**：直接由浏览器前端发起请求，受安全策略限制较多。
- **代理行为**：支持「放宽证书校验」（用于自签名证书）和「强制 HTTP/1.1」（提高与旧版 API 或某些代理的兼容性）。
- **自定义请求头**：可为该渠道所有请求附加特定的 HTTP Header。
- **高级端点**：允许覆盖特定操作（如 `chat/completions` 或 `embeddings`）的具体路径。

## 步骤三：管理 API Key 与负载均衡

AIO Hub 支持**多密钥管理**，能够有效应对速率限制（Rate Limit）。

1. **快速输入**：在密钥框中输入，多个 Key 请使用英文逗号分隔。
2. **状态管理弹窗**：点击「管理密钥状态」进入高级管理界面：
   - **健康度监控**：实时查看每个 Key 的错误计数和最后报错信息。
   - **自动熔断**：开启后，若某个 Key 报错（如 429），系统将自动暂时禁用。
   - **自动恢复**：可设置在指定分钟后自动尝试重新启用已熔断的 Key。
   - **批量操作**：支持批量导入、导出、测试连通性以及一键清理失效密钥。

---

## 步骤四：深度模型配置

模型管理是渠道配置的核心。点击「添加模型」或「编辑」图标进入详细配置：

### 1. 基本信息与预设

- **模型 ID**：API 调用时的真实 ID。
- **应用预设**：输入 ID 后点击「应用预设」，系统将自动匹配并填充名称、分组、图标和模型能力。

### 2. 模型能力开关

通过开关精确控制该模型在 UI 中展现的功能：

- **文本生成 / 流式输出**：基础对话能力。
- **图片输入 (Vision)**：是否支持视觉分析。
- **工具调用 (Function Calling)**：是否支持调用外部插件。
- **思考能力 (Reasoning)**：针对深度思考模型（如 o1, r1），可配置配置模式（开关、预算或等级模式）。

### 3. Token 限制与价格

- **上下文窗口**：配置模型支持的最大 Token 总数。
- **输出限制**：限制单次响应的最大生成长度。
- **价格配置**：设置输入、输出、请求及图像的单价，用于对话时的成本预估。

### 4. 高级配置

- **自定义参数**：使用 JSON 格式为该模型指定非标准的专属 API 参数（如特定的 `top_p` 或供应商专属字段）。
- **后处理规则**：配置该模型默认开启的后处理逻辑（如自动提取代码块、思维链过滤等）。

---

## 连接测试

- **一键测试**：点击 API Key 旁的「测试连接」，系统将使用配置的第一个模型进行快速握手。
- **单模型测试**：在模型列表中点击测试图标，验证特定模型的可用性。
- **单密钥测试**：在多密钥管理弹窗中，针对特定 Key 进行连通性验证。

## 常见问题

### Q：配置完成后在哪里使用 AI 功能？

在 **LLM 智能对话** 工具中，选择你配置好的渠道和模型即可开始对话。

### Q：多 API Key 的负载均衡策略是什么？

系统采用**轮询策略**。如果当前使用的 Key 请求失败且触发了熔断，系统会自动切换到下一个健康密钥重试。

### Q：为什么我的模型无法上传图片？

请检查该模型的「模型能力」配置中是否开启了「图片输入」开关。
