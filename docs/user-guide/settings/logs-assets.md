# 日志与资产管理

> 日志设置让你精确控制应用的日志记录行为；资产管理则管理所有文件类资产的存储位置。本篇将带你了解如何查询日志、调整日志级别，以及配置资产存储路径和工具可见性。

---

## 日志设置

1. 点击标题栏右侧的 ⚙️ 图标进入设置。
2. 在设置左侧导航栏中点击「日志配置」选项卡。

### 日志级别

| 级别             | 说明                     | 适用场景               |
| :--------------- | :----------------------- | :--------------------- |
| **DEBUG (调试)** | 记录所有日志消息，最详细 | 问题排查、开发调试     |
| **INFO (信息)**  | 记录常规操作信息         | 日常使用（推荐）       |
| **WARN (警告)**  | 仅记录警告和错误         | 生产环境，减少日志量   |
| **ERROR (错误)** | 仅记录错误信息           | 最小日志量，仅关注异常 |

> **实时生效**：切换日志级别后，系统立即使用新级别记录，无需重启应用。

### 输出控制

| 配置项         | 说明                                                                  |
| :------------- | :-------------------------------------------------------------------- |
| **文件日志**   | 将日志保存到本地文件 `appDataDir/logs/app-YYYY-MM-DD.log`，按日期分割 |
| **控制台日志** | 在浏览器开发者控制台（DevTools）中显示日志                            |

两个开关互不影响，你可以：

- 只存文件不看控制台（减少 DevTools 噪音）
- 只看控制台不存文件（节省磁盘空间）
- 两个都开（开发和排查问题）

### 缓冲区配置

| 配置项             | 可选值                                        | 说明                                       |
| :----------------- | :-------------------------------------------- | :----------------------------------------- |
| **日志缓冲区大小** | 500 / 1000 / 2000 / 5000 / 10,000 / 50,000 条 | 内存中保留的日志条数，超过后删除最早的日志 |
| **文件分割阈值**   | 1MB / 2MB / 5MB / 10MB / 20MB                 | 当日志文件超过此大小，自动分割创建新文件   |

> 缓冲区大小仅影响内存中的日志，不影响已保存的日志文件。

### 日志统计

配置面板中有一个实时更新的统计区域，以卡片形式展示各类别日志条数：

```
┌───────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
│  总计 │ │ DEBUG│ │ INFO │ │ WARN │ │ ERROR│
│  128  │ │  42  │ │  68  │ │  14  │ │   4  │
└───────┘ └──────┘ └──────┘ └──────┘ └──────┘
```

每个卡片左侧有颜色标识（主题色/灰色/绿色/橙色/红色），一目了然。

### 日志操作

| 操作             | 说明                                                                 |
| :--------------- | :------------------------------------------------------------------- |
| **打开日志目录** | 打开日志文件所在的文件夹（通过 Rust 后端强制绕过前端路径限制）       |
| **导出日志**     | 将内存中的日志保存为 TXT 文件                                        |
| **清空缓冲区**   | 清空内存中的日志缓冲区，不影响已保存的文件                           |
| **测试日志**     | 生成一条测试日志（DEBUG / INFO / WARN / ERROR 各一条），便于验证配置 |

### 当前日志文件

面板底部显示当前正在写入的日志文件完整路径，方便你直接定位：

```
appDataDir/logs/app-2025-10-01.log
```

## 资产管理

资产管理位于通用设置面板中，用于配置应用内所有文件类资产的存储位置。

> **什么是资产？** 资产包括 LLM 聊天中的附件（图片、文档）、OCR 处理的图片、媒体生成器产生的文件等。

### 路径说明

| 项               | 说明                                                   |
| :--------------- | :----------------------------------------------------- |
| **默认路径**     | 应用数据目录下的 `assets` 文件夹（通过 Rust 后端获取） |
| **自定义路径**   | 你自己指定的存储位置，为空时使用默认路径               |
| **当前使用路径** | 实际生效的路径（自定义有值则用自定义，否则用默认）     |

### 操作指南

**设置自定义路径**：

1. 点击「选择目录」按钮
2. 在弹出的目录选择器中指定新的存储位置
3. 设置会自动保存并生效
4. 同时会清除 `useAssetManager` 中的路径缓存

**重置为默认**：

- 如果已经设置了自定义路径，点击「重置为默认」按钮即可恢复

**打开目录**：

- 点击「打开目录」按钮快速跳转到当前资产存储位置

### 重要说明

| 事项                     | 说明                                               |
| :----------------------- | :------------------------------------------------- |
| **仅影响新导入的资产**   | 修改路径后，已有资产**不会**自动迁移到新位置       |
| **需要手动迁移**         | 如果确实需要移动已有资产，请手动操作               |
| **按类型和日期自动分类** | 资产自动按类型和日期分类存储，如 `images/2025-10/` |
| **修改后立即生效**       | 下次资源加载时使用新路径                           |

## 常见问题

### Q：日志文件会占用很多磁盘空间吗？

日志模块提供了文件分割和级别控制两种方式管理磁盘占用：

1. 设置较高的日志级别（如 WARN 或 ERROR）减少日志量
2. 文件分割阈值设为较小值（如 1MB），超过自动分割，便于清理
3. 日志按日期分割，你可以定期清理旧的日志文件

### Q：修改资产路径后，之前导入的文件还能用吗？

**能。** 修改路径只会影响新导入的资产。已有资产仍然在原路径下，应用仍然可以正常访问。如果你希望新旧资产在一起，需要手动搬迁文件。
