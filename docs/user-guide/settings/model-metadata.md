# AI 模型元数据配置

> 这个子面板用于控制**模型在 UI 中的图标和分组显示**。系统通过规则匹配自动为模型分配图标，帮助你区分不同提供商的各种模型家族。

---

## 入口

1. 点击标题栏右侧的 ⚙️ 图标进入设置。
2. 在左侧导航栏中选择 **AI 模型元数据配置**。

---

## 规则匹配逻辑

系统通过规则引擎自动为模型分配图标。规则按优先级排序，匹配成功后立即生效。

### 规则匹配类型

| 类型         | 匹配方式           | 示例     |
| :----------- | :----------------- | :------- |
| **Provider** | 匹配服务提供商名称 | `openai` |
| **Model**    | 精确匹配模型 ID    | `gpt-4o` |
| **Prefix**   | 匹配模型 ID 前缀   | `gpt-`   |
| **Group**    | 匹配分组名称       | `chat`   |

支持**正则表达式**。

---

## 操作界面

### 视图与过滤

- **网格/列表** 双视图切换。
- **搜索过滤**：按关键字（匹配值、类型、分组、描述）、启用状态搜索。
- **排序**：按优先级、类型、名称、创建时间。
- **分页**：管理大量规则时保持界面清晰。

### 常用操作

| 操作                 | 说明                                   |
| :------------------- | :------------------------------------- |
| **添加规则**         | 选择匹配类型，输入匹配值，选择预设图标 |
| **编辑/删除**        | 修改已有规则                           |
| **启用/禁用**        | 临时关闭某条规则而不删除               |
| **重置为默认**       | 恢复内置规则                           |
| **合并最新内置规则** | 在应用更新后，合并新版内置规则         |
| **导入/导出 JSON**   | 批量管理规则集                         |

---

## 测试模式

在测试模式下，输入**模型 ID + Provider**，系统会实时展示：

- 匹配状态和命中的规则
- 匹配到的图标预览
- 候选规则列表（用于调试未匹配的情况）

这在你新增模型后想确认图标是否正确时非常有用。

---

## 常见问题

### Q：模型元数据规则的作用是什么？

它控制模型在编辑器和对话框中的显示图标和分组，对于区分同一提供商的不同模型家族很有用（如分清 GPT-4o 和 o1 系列）。
