# 用户档案管理

用户档案用于描述「你是谁」。在 LLM Chat 中，它会作为用户身份信息参与上下文构建，让不同 Agent 能以更准确的口吻、称呼和偏好与你对话。

## 入口

你可以从两个位置进入用户档案管理：

- 设置 -> 用户档案管理。
- 标题栏右侧的用户档案入口 -> 管理用户档案。

标题栏中选择的档案会作为全局用户档案；某个 Agent 也可以单独关联自己的用户档案。

## 新建用户档案

1. 进入「用户档案管理」。
2. 点击新建按钮。
3. 填写 ID/名称、显示名称和描述。
4. 选择头像。
5. 按需关联世界书和快捷操作。
6. 保存后，在标题栏或 Agent 编辑器中选择该档案。

建议先写一个简短稳定的基础档案，再为不同场景创建专用档案。

## 字段说明

| 字段         | 说明                                     |
| :----------- | :--------------------------------------- |
| ID/名称      | 档案的内部名称，也会用于部分宏替换场景   |
| 显示名称     | 界面中显示的名称；不填时使用 ID/名称     |
| 头像         | 支持上传图像、选择历史头像或使用路径     |
| 描述         | 注入给 AI 的用户身份信息                 |
| 关联世界书   | 当前档案生效时附带的世界书               |
| 快捷操作     | 当前档案生效时显示的快捷操作组           |
| 消息样式     | 用户消息是否跟随 Agent，或使用自定义样式 |
| 文本替换规则 | 针对用户消息的正则预处理规则             |

## 如何写好描述

描述不是越长越好。它应该告诉 AI 与当前对话强相关的信息。

推荐结构：

```text
我是：一句话身份。
偏好：喜欢的称呼、语气和回答粒度。
禁忌：不希望 AI 做的事。
上下文：长期有效的背景信息。
```

示例：

```text
我是一个独立开发者，主要维护桌面端 AI 工具。
我喜欢简洁但准确的技术说明，遇到风险请直接指出。
默认用中文回答，代码和命令保留英文。
```

## 全局档案与 Agent 档案

用户档案有两个常见生效来源：

- 全局用户档案：通过标题栏选择，适用于大多数会话。
- Agent 关联档案：在 Agent 编辑器中指定，适用于特定角色或场景。

当 Agent 有自己的用户档案时，系统会优先使用 Agent 配置；否则使用全局档案作为兜底。

## 关联世界书

如果某个用户档案需要长期背景设定，可以关联世界书。例如：

- 你的项目背景。
- 角色扮演中的玩家设定。
- 常用术语和人际关系。
- 写作世界观中和「用户」相关的设定。

关联后，当对话触发关键词时，世界书条目会自动注入上下文。

## 关联快捷操作

用户档案可以关联快捷操作组。这样当你切换到某个档案时，输入框工具栏会显示对应的常用指令。

例如：

- 开发者档案：代码审查、生成测试、解释报错。
- 写作者档案：润色、续写、检查人物口吻。
- 学习档案：生成练习题、总结重点、追问概念。

## 消息样式与文本替换

如果你希望某个用户档案有专属发言样式，可以把消息样式切为「自定义」，再调整 Markdown 样式。

文本替换规则适合做轻量预处理，例如：

- 自动修正常用错别字。
- 把固定简称展开为完整描述。
- 为特定口癖或角色表达做格式化。

正则规则会影响消息进入上下文前的文本，请先用小范围规则测试。

## 数据存储与删除

每个用户档案会以独立目录保存，包含 `profile.json` 和头像等资源。删除档案时，目录会尽量移入系统回收站。

建议在大量编辑前先导出或备份配置目录，尤其是包含头像和复杂样式的档案。

## 常见问题

### 为什么 AI 没有体现我的用户档案？

检查当前标题栏是否选中了该档案，或者当前 Agent 是否绑定了另一个档案。也要确认 Agent 的提示词中保留了用户档案插入位置。

### 用户档案会覆盖 Agent 人设吗？

不会。用户档案描述的是用户侧身份，Agent 人设描述的是 AI 侧角色。两者会共同进入上下文。

### 描述写很长会有问题吗？

会增加 Token 占用，也可能稀释重点。长期背景建议放到世界书，用户档案只保留稳定而核心的信息。

### 删除档案后还能恢复吗？

如果成功移入系统回收站，可以从回收站恢复。若回收站不可用或被清空，则需要依赖你自己的备份。
