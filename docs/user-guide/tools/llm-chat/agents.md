# 🤖 智能体 (Agent)

智能体 (Agent) 是 LLM Chat 的核心概念——它是**角色设定 + 模型参数 + 工具权限 + 预设消息**的完整配置包。你可以在不同 Agent 之间快速切换，以适应不同的对话场景。

## Agent 的概念

一个典型的 Agent 包含：

| 组件        | 说明                                              |
| ----------- | ------------------------------------------------- |
| 🆔 基本信息 | 名称、图标、描述                                  |
| 👤 角色设定 | 系统提示词、角色人格定义                          |
| ⚙️ 模型参数 | 关联的服务渠道 (Profile)、模型 ID、Temperature 等 |
| 📝 预设消息 | 上下文预设消息（系统提示、Few-shot 示例等）       |
| 🔧 工具配置 | 工具调用开关、自动批准规则                        |
| 📖 关联资源 | 世界书、快捷操作、用户档案绑定                    |
| 🧩 扩展配置 | 插件扩展启用状态                                  |

## 创建 Agent

### 从零创建

1. 在左侧 Agent 列表顶部点击 ➕ 按钮
2. 填写 Agent 名称（必填）和描述
3. 选择使用的服务渠道和模型
4. 可选：添加系统提示词和预设消息
5. 点击确认

### 复制已有 Agent

右键点击 Agent 条目选择「复制」，可以创建该 Agent 的完整副本，包括其资产文件目录。

### 从 SillyTavern 导入

支持导入 SillyTavern 格式的角色卡文件（PNG/JSON）：

1. 在创建 Agent 时选择「导入 SillyTavern 角色卡」
2. 选择文件后，系统会自动解析角色信息
3. 导入的角色包含预设消息、参数配置和资产文件

## Agent 编辑器

编辑 Agent 时，采用左侧导航、右侧表单的分段式编辑器。

### 1. 基础设定 (Basic)

- **ID/名称**：Agent 的唯一内部标识。
- **显示名称**：UI 中展示的友好名称。
- **图标**：支持图片、Emoji 或内置图标。
- **分类与标签**：用于在列表中进行筛选和组织。
- **描述**：对 Agent 功能或背景的简要介绍。

### 2. 角色设定 (Personality)

这是 Agent 的灵魂所在，包含模型、预设消息、知识库、世界书和快捷操作的综合配置。

- **模型**：为该 Agent 指定默认的服务渠道（Profile）和模型。
- **关联用户档案**：绑定特定的 [用户档案](./user-profiles)，自动注入用户背景。支持快速打开档案管理。
- **快捷操作**：关联 [快捷操作](./quick-actions) 组，激活此 Agent 时快捷操作显示在输入框工具栏中。
- **关联世界书**：选择要关联的 [世界书](./worldbook)，对话匹配到关键词时自动注入相关设定。
  - **世界书高级设置**（可折叠展开）：
    - **禁用递归扫描**：开启后，世界书条目触发后不再扫描其内容中的关键词。
    - **默认扫描深度**：回溯多少条历史消息进行关键词匹配（0~100，默认 3）。
- **预设消息 (Presets)**：定义对话的"前传"或 Few-shot 示例。
  - 支持 `system` / `user` / `assistant` 角色。
  - **注入策略**：支持深度注入（Depth）、高级深度配置（`advanced_depth`，支持多点/循环语法如 `3, 10~5`）或锚点注入（Anchor），精准控制消息在上下文中的位置。
  - **显示数量**：设置在聊天界面中作为开场白显示的预设消息条数（0~16，0 表示不显示）。
- **知识库 (RAG)**：配置关联的知识库检索参数（可折叠展开的「知识库高级设置」面板）：
  - **默认检索引擎**：`vector`（向量）/ `fulltext`（全文）/ `hybrid`（混合）/ `blender`（混合排序）。
  - **默认召回数量**：每次检索召回的知识片段数（1~50）。
  - **召回字数上限**：检索结果总字数上限，0 表示不限制。
  - **Embedding 模型**：用于向量检索的专用模型（如不设置则无法使用向量引擎）。
  - **默认最低分数**：低于此相关度分数的知识片段被过滤（0.0~1.0）。
  - **结果模板**：检索结果注入提示词的格式模板，支持 `{count}`, `{kbName}`, `{key}`, `{content}`, `{score}`, `{tags}` 等变量。
  - **空结果提示**：未检索到内容时的占位文本。
  - **门控扫描深度**：标签门控模式下扫描最近多少条消息匹配关键词（1~20）。
  - **上下文窗口**：用于构建检索查询的最近用户消息数量（1~10）。
  - **启用检索缓存**：开启后相似查询复用缓存结果。
  - **缓存相似度阈值**：余弦相似度高于此值时命中缓存（0.8~1.0）。
  - **查询衰减因子**：多轮查询聚合时的权重衰减（0.1~1.0）。
  - **启用结果聚合**：聚合当前结果与历史轮次的结果。
  - **结果衰减因子**：历史检索结果的分数衰减因子（0.1~1.0）。
  - **最大聚合轮次**：结果聚合时保留的最大历史对话轮次（1~10）。

### 3. 功能扩展 (Capabilities)

- **资产管理**：管理 Agent 私有的媒体文件（图片、音频等），支持通过 `agent-asset://` 协议在对话中引用。
  - 支持按分组组织资产（如"表情包""背景音乐"等），每个分组可设置名称、图标和排序。
  - 资产创建后才可管理（创建模式下按钮不可用）。
- **虚拟时间线**：为角色扮演设定独立的虚拟时钟和时间流速。
  - **虚拟基准点**：虚拟世界的起始时间。
  - **现实基准点**：对应现实世界的时刻（支持"设为现在"快捷操作）。
  - **时间流速倍率**：虚拟时间相对于现实时间的流逝速度（如 2.0 意为虚拟时间比现实快一倍）。
  - **宏实时预览**：输入 `{{time}}`、`{{datetime_cn}}`、`{{shichen}}` 等宏，实时查看虚拟时间下的展开结果。预览每秒自动刷新。

### 4. 会话变量 (Variables)

允许定义 Agent 专属的动态变量。这些变量可以在提示词中通过 `{{var_name}}` 引用，并支持在对话过程中通过宏动态修改。

### 5. 工具调用 (Tool Calling)

配置 Agent 访问外部工具的权限。启用后可通过 `{{tools}}`、`{{tool_usage}}`、`{{tool_context}}` 宏在提示词中注入工具定义和上下文。

**基础配置**：

- **启用/禁用**：顶部开关控制工具调用的总开关。
- **自动批准**：切换「自动」或「手动」模式。自动模式下白名单内的工具可静默执行。
- **角色转换**：开启后工具执行结果以"用户"身份发送给模型，适用于不支持工具角色的纯文本协议（如 VCP）。
- **保底注入**：开启后如果提示词中未手动添加 `{{tools}}` 宏，系统自动将工具定义注入到对话历史之前。
- **并行执行**：是否允许多个工具同时执行。
- **最大迭代**：单次工具调用链的最大迭代次数（1~10，默认 20）。_注：UI 滑块上限 10，代码默认 20。_
- **超时 (ms)**：每次工具调用的超时时间（默认 30000ms，步进 5000ms）。

**工具列表管理**：

- **搜索过滤**：按工具名称、ID、方法或描述实时搜索。
- **显示方法统计**：开关控制是否在工具条目中显示方法数量。
- **默认启用**：新发现工具是否默认启用。
- **默认自动批准**：新发现工具是否默认开启自动批准。
- **配置复制/粘贴**：批量复制/粘贴所有工具的自定义配置（JSON 格式），便于在不同 Agent 间迁移设置。
- **每个工具可展开的详细配置**：包含方法级开关、自动批准、描述覆盖、自定义设置等。

### 6. 环境增强 (Environment)

通过插件扩展上下文能力。启用后相关信息通过 `{{tool_context}}` 宏注入到提示词中。

- **总开关**：顶部开关控制是否启环境增强。
- **默认启用新插件**：新发现的环境增强插件是否默认启用。
- **可用插件列表**：列出已发现的环境增强插件，每个插件可独立开关。插件提供额外的运行时上下文信息（如系统状态、剪贴板内容、当前时间等）。

### 7. 输出与显示 (Output)

控制 Agent 的交互和视觉表现：

- **分支发送模式**：开启后，点击由 LLM 生成的交互按钮（如 RPG 选项）时，将从此消息下方创建新分支而非追加到末尾。
- **工具调用折叠**：开启后消息中的工具调用组件默认处于折叠状态。
- **媒体音量**：调节该 Agent 输出音频的初始音量百分比（0~100）。最终音量 = 原始内容音量 × 全局音量 × Agent 音量。
- **文本替换规则 (Regex)**：配置该 Agent 专属的正则替换规则，用于对消息内容进行动态清洗或格式转换。支持折叠展开编辑。
- **思考块规则配置**：自定义 LLM 输出中思考过程（如 `<think>` 标签）的识别和显示规则。支持折叠展开编辑。
- **回复样式自定义**：自定义该 Agent 的 Markdown 渲染样式（如粗体颜色、发光效果等）。支持折叠展开编辑。
- **视觉化输出指南**：提供 HTML/CSS/JS 规范，引导模型输出结构化的视觉内容。独立的文本编辑器，支持 `{{char}}` 等宏在指南中引用。

## 切换 Agent

- **左侧面板**：点击 Agent 列表中的条目即可切换
- **快速切换**：在输入框上方使用 🤖 下拉菜单快速切换
- **自动切换**：切换到某个会话时，可设置自动切换为其使用的 Agent

## Agent 数据隔离

每个 Agent 独立存储数据，包括：

- 配置文件和预设消息
- 关联的资产文件目录
- 用户自定义的样式配置

Agent 删除时，其目录和文件会被移入系统回收站。
