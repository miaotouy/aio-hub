# Agent 编辑器指南

Agent 编辑器是定义智能体行为、外观和能力的核心界面。它采用左侧导航、右侧表单的分段式结构，确保即使在配置项极多时也能保持清晰。

## 1. 基础设定 (Basic)

管理 Agent 的身份标识和组织方式：

- **ID/名称**: Agent 的唯一内部标识符，用于数据存储。
- **显示名称**: 在聊天界面和列表中展示的友好名称。
- **图标**: 支持上传本地图片、选择 Emoji 或使用系统内置图标库。
- **分类与标签**: 通过标签对 Agent 进行归类，方便在大型 Agent 库中快速检索。
- **描述**: 对 Agent 的功能、背景或使用建议的简要介绍。

## 2. 创建与导入

### 从零创建

1. 在左侧 Agent 列表顶部点击 **➕ (创建新智能体)** 按钮。
2. 填写基础信息并选择默认的服务渠道与模型。
3. 根据需要配置系统提示词（System Prompt）。

### 复制已有 Agent

右键点击 Agent 条目选择 **「复制」**。系统会创建一个包含所有配置、预设消息以及私有资产目录完整副本的新 Agent。

### 导入 SillyTavern 角色卡

系统深度兼容 SillyTavern 格式（PNG 角色卡或 JSON）：

1. 在创建界面选择 **「导入 SillyTavern 角色卡」**。
2. 导入后，系统会自动解析并填充：名称、描述、性格设定、场景描述、开场白（预设消息）以及关联的资产图片。

## 3. 环境增强 (Environment)

通过插件扩展 Agent 的感知能力。启用后，插件提供的信息将通过 `{{tool_context}}` 宏注入到提示词中。

- **总开关**: 控制环境增强功能的全局启用。
- **可用插件列表**:
  - **系统状态**: 注入当前 CPU、内存、电池等硬件状态。
  - **剪贴板**: 获取最新的剪贴板文本内容。
  - **运行时上下文**: 注入当前时间、日期、操作系统版本等信息。
- **默认策略**: 可设置新发现的插件是否默认启用。

## 4. 输出与显示 (Output)

控制 Agent 的交互行为和视觉渲染风格：

- **分支发送模式**: 开启后，点击 LLM 生成的交互按钮（如 RPG 游戏中的选项）时，系统会从该消息节点创建新分支，而不是在对话末尾追加，非常适合多分支剧情探索。
- **工具调用折叠**: 开启后，繁琐的工具调用过程（JSON 参数和原始返回结果）在聊天流中默认折叠。
- **媒体音量**: 为该 Agent 设置独立的音量倍率，平衡不同角色的语音输出大小。
- **文本替换规则 (Regex)**: 允许配置 Agent 专属的[正则管道](../context-pipeline/regex-pipeline)，用于实时清洗输出内容。
- **思考块规则**: 配置如何识别和展示 LLM 的思考过程（如 `<think>` 标签内容）。
- **回复样式自定义**: 支持通过 CSS 变量自定义 Markdown 渲染效果（如发光文字、特定角色主题色）。
- **视觉化输出指南**: 一个专用的文本编辑器，用于存放 HTML/CSS/JS 渲染规范，引导模型输出符合 UI 框架要求的结构化视觉内容。

## 5. 用户档案与会话变量

- **关联用户档案**: 绑定特定的[用户档案](./user-profiles)，使 Agent 能够感知“你是谁”。
- **会话变量 (Variables)**: 定义动态变量（如“好感度”、“金钱”），并在提示词中通过 `{{var_name}}` 引用。详见[变量系统](../macro-system/session-variables)。
