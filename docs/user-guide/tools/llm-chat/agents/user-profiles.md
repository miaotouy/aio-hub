# 用户档案 (User Profiles)

用户档案用于存储关于“你”（用户）的信息。通过将档案与 Agent 绑定，Agent 可以在对话中感知你的身份、偏好和背景，从而提供更具代入感的回复。

## 1. 核心作用

- **身份注入**: 自动将你的姓名、性别、年龄等基本信息注入提示词。
- **背景感知**: 存储你的职业、爱好、过往经历等长效背景。
- **动态引用**: 在提示词中通过 `{{user}}` 宏引用档案中的姓名，通过 `{{user_persona}}` 引用详细描述。

## 2. 管理档案

### 创建与编辑
1. 在 Agent 编辑器的「角色设定」面板中，点击 **「关联用户档案」**。
2. 在弹出的对话框中，你可以创建多个不同的档案（如“工作模式”、“游戏模式”）。
3. 填写详细的描述（Persona），描述你希望 Agent 如何看待你。

### 档案组件
- **姓名**: 你的显示名称。
- **描述**: 关于你的详细设定。
- **头像**: 支持图片或 Emoji。

## 3. 绑定策略

每个 Agent 只能绑定一个活跃的用户档案。
- **全局默认**: 你可以在系统设置中指定一个全局默认档案。
- **Agent 覆盖**: 在 Agent 编辑器中指定的档案将覆盖全局设置。

## 4. 预览与调试

你可以通过[上下文分析器](../context-pipeline/analyzer)查看用户档案是如何被组装进最终发送给 LLM 的提示词中的。通常，档案内容会作为系统提示词的一部分或特定的 `user` 角色消息注入。
