# 📎 附件与资产

LLM Chat 支持将图片、音频、视频、文档等文件作为附件随消息一起发送，并深度集成资产管理器和多模态转写系统。

## 支持的附件类型

| 类型    | 格式                          | 说明                             |
| ------- | ----------------------------- | -------------------------------- |
| 📷 图片 | PNG, JPG, GIF, WebP, SVG, BMP | 自动生成缩略图预览，支持拖拽排序 |
| 🎵 音频 | MP3, WAV, OGG, M4A, AAC, FLAC | 自动转写为文本（需启用转写功能） |
| 🎬 视频 | MP4, AVI, MOV, MKV, WebM      | 自动压缩和转写                   |
| 📄 文档 | PDF, TXT, DOCX, MD, CSV, JSON | PDF 自动转写，纯文本直接插入     |
| 📁 其他 | 任何文件                      | 作为文件引用发送                 |

## 添加附件

### 方法一：点击按钮

1. 在输入框上方找到 📎 按钮
2. 点击打开文件选择器
3. 选择一个或多个文件
4. 文件会立即显示为预览卡片，后台异步导入

### 方法二：拖拽上传

直接将文件从文件管理器拖拽到对话区域或输入框上方的拖放区域即可。

### 方法三：粘贴

从剪贴板复制图片等文件，在输入框中粘贴即可自动添加。

### 方法四：从资产管理器

在资产管理器中选择文件，点击「发送到 LLM Chat」按钮。

## 附件处理流程

为了保证丝滑的操作体验，LLM Chat 采用了**非阻塞异步处理**架构：

1. **瞬时预览**：添加文件后，UI 会立即显示预览卡片，不会因为文件大而卡顿。
2. **后台静默导入**：系统在后台自动完成文件复制、SHA256 去重和缩略图生成。
3. **智能能力感知**：系统会自动识别当前所选模型的能力（如是否支持 Vision、音频或长文档）。
   - 如果模型不支持，卡片会显示警告，提示你切换模型或开启转写。
4. **自动引用转换**：发送消息时，系统会自动将文件转换为模型可理解的 Base64 或云端 URL。

### 导入状态指示

| 状态      | 指示     | 说明                     |
| --------- | -------- | ------------------------ |
| ⏳ 等待中 | 灰色边框 | 正在排队等待导入         |
| 🔄 导入中 | 旋转动画 | 正在文件复制和缩略图生成 |
| ✅ 完成   | 正常显示 | 导入成功，已就绪         |
| ❌ 失败   | 红色边框 | 导入出错，可尝试重新添加 |

## 多模态转写

如果当前模型不支持某个附件类型，系统会自动利用你的 AI 服务进行**转写**——将图片/音频/视频/文档的内容提取为文本，然后再发送给模型。

### 转写策略

- **智能模式（默认）**：模型原生支持该类型就直接发送文件，不支持则自动转写
- **总是转写**：无论模型是否支持，都强制转写所有附件

### 转写配置

在聊天设置 → 附件转写中可以详细配置：

- 为不同附件类型指定专门的转写模型
- 自定义转写 Prompt
- 控制转写的精细度（温度、最大 Token）
- 配置视频/音频压缩参数（需要 FFmpeg）

## 管理附件

添加附件后，你可以：

- **查看预览**：图片显示缩略图，音频显示播放器
- **移除附件**：点击附件卡片上的 ✕ 按钮
- **拖拽排序**：调整多附件时的顺序
- **查看提示**：如果模型不支持该类型，卡片上会显示警告信息

## 与资产管理器的集成

所有添加的附件都会通过资产管理系统的导入管道处理。导入成功的文件会进入资产管理器的索引，你可以在资产管理器中对它们进行：

- 浏览和管理历史附件
- 按来源模块筛选
- 批量操作（删除、导出）

## 输入框中的附件占位符

当你发送包含附件的消息时，系统会在消息文本中插入类似 `【file::assetId::文件名】` 的占位符。上下文管道中的 `Asset Resolver` 处理器会在发送前将这些占位符解析为模型能够理解的文件引用或内容。
