# 上下文分析器 (Context Analyzer)

上下文分析器是一个强大的调试工具，它允许您在消息发送给 LLM 之前，查看并分析经过“上下文后处理管道”处理后的最终状态。它可以帮助您理解宏是如何展开的、世界书是如何注入的、以及 Token 是如何分布的。

## 1. 核心功能

上下文分析器提供五个维度的深度视图：

### 结构化视图 (Structured View)
- **智能体快照**：显示当前使用的智能体名称、模型和配置信息。
- **统计面板**：提供 Token 和字符的详细分布，包括预设消息、会话历史、后处理消耗和世界书消耗。
- **消息溯源**：标记每条消息的来源（预设、历史、注入），并显示其 Token 消耗和字符数。
- **附件分析**：显示消息中包含的附件及其 Token 消耗，并标记是否使用了转写。

### 原始请求 (Raw Request)
- **API 模拟**：以标准 JSON 格式展示最终发送给模型供应商的请求体。
- **参数展开**：显示所有模型参数（如 Temperature, Top-P 等）和自定义参数的最终值。
- **消息序列**：展示经过管道处理后的纯净消息数组。

### 内容分析 (Analysis Chart)
- **可视化占比**：通过饼图直观展示不同部分（预设、历史、世界书、后处理）的 Token 或字符占比。
- **详细数据表**：提供精确的数值和百分比，帮助您优化长文本上下文。

### 宏调试 (Macro Debug)
- **宏检测**：自动识别文本中使用的所有宏（如 `{{time}}`, `{{tools}}` 等）。
- **执行预览**：显示宏展开后的实时结果。
- **替换对比**：提供类似 Git diff 的差异视图，对比宏展开前后的文本变化。

### 变量状态 (Variables View)
- **快照追踪**：显示会话中通过 `<svar />` 标签定义的变量及其当前值。
- **路径与类型**：展示变量的完整路径（如 `character.hp`）及其数据类型。

## 2. 激活世界书分析

当世界书条目被激活并注入上下文时，分析器会显示专门的“激活的世界书条目”区域：
- **位置标记**：显示条目被注入的具体位置（如：角色设定前、作者注释后、指定深度等）。
- **关键词匹配**：展示触发该条目注入的关键词。
- **内容预览**：查看注入到上下文中的原始文本。

## 3. 使用场景

- **调试宏错误**：当宏展开不符合预期时，通过“宏调试”视图查看原始输入与处理后的差异。
- **优化 Token 消耗**：通过“内容分析”发现哪个部分占用了过多上下文，从而调整截断策略或压缩配置。
- **验证管道逻辑**：确认正则替换、世界书注入和上下文压缩是否按预期工作。
- **模拟请求排错**：如果 API 报错，可以通过“原始请求”视图复制 JSON 内容，在外部工具（如 Postman）中进行复现。
