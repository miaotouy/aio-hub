# 📤 导出与导入

LLM Chat 支持将会话、分支和 Agent 导出为多种格式，也支持从外部文件导入数据。

## 会话导出

### 导出为 Markdown

**会话导出**：在右侧会话列表右键点击会话 → 「导出会话」，可选择格式并预览导出内容。

**分支导出**：在消息的操作菜单中点击「更多」→「导出分支」，可从此节点开始导出从根节点到该消息的路径。支持丰富的 Markdown 格式：

```
# 对话记录: 会话名称

导出时间：2026-05-08 10:30:00
消息统计：共 12 条消息
对话参与者：用户: 小林 | 智能体: 助手

---
## 🧑 小林 (10:30)
你好！

## 🤖 助手 (10:31)
你好！有什么可以帮你的？
```

导出内容包括：

- 会话基本信息（名称、创建时间）
- 参与者和智能体信息
- 每条消息：发送者、时间戳、内容
- Token 使用统计
- 模型信息
- 附件列表

**完整会话树导出**：导出包含所有分支的完整会话，使用 Markdown 列表树状结构展示：

```
- **🤖 助手** (10:31)
  这是第一条回复
  *[此处产生了 2 条不同的对话路径]*

  - **🧑 小林** (10:32)
    追问内容...

  - **🧑 小林** (10:33)
    另一个方向的追问...
```

### 导出为 JSON

分支导出也支持 JSON 格式，保留完整的结构化数据，适合程序化处理或数据备份：

```json
{
  "session": {
    "name": "会话名称",
    "createdAt": "2026-05-08T02:30:00.000Z"
  },
  "exportTime": "2026-05-08T10:30:00.000Z",
  "messageCount": 12,
  "messages": [...]
}
```

### 导出选项

导出时可以从对话框中选择：

- **包含预设消息**：是否包含 Agent 的预设消息
- **合并预设**：将预设消息与会话消息合并显示，或分开显示
- **包含用户档案信息**：参与者名称和头像
- **包含智能体信息**：使用的 Agent 名称
- **包含模型信息**：使用的模型和渠道
- **包含 Token 使用统计**
- **包含附件列表**
- **包含错误信息**
- **范围选择**：只导出指定范围内的消息

## Agent 导出

### 导出 Agent 配置

在智能体编辑器中可以将 Agent 导出为共享文件：

- 支持 JSON 格式
- 可选择是否包含资产文件（头像、附件等）
- 支持 ZIP 打包（包含资产目录）
- 支持 PNG 格式（SillyTavern 兼容的角色卡）

### 导出代理应用

导出的 Agent 文件可以在设备之间迁移，或分享给其他用户。

## 导入

### 导入会话

目前支持从文件加载之前保存的完整会话备份。

### 导入 Agent

1. 在 Agent 管理中选择「导入 Agent」
2. 选择要导入的文件（JSON / ZIP / PNG 角色卡）
3. 系统会自动**预检**：检查冲突的名称、模型兼容性等
4. 解决冲突后确认导入

**预检详情**：

- 检测与现有 Agent 的名称冲突
- 检查使用的模型在本地是否可用
- 列出导入的资产文件列表
- 显示包含的预设消息数量

### 从 SillyTavern 导入角色卡

支持导入 SillyTavern 格式的 PNG 角色卡（V2 格式）：

1. 在创建 Agent 时选择「导入 SillyTavern 角色卡」
2. 选择 `.png` 文件
3. 系统会自动解析角色信息（名称、描述、预设消息）和嵌入的资产文件

### 批量导入

导入功能支持单个文件和批量文件同时处理。

## 数据迁移

### 跨设备迁移

推荐使用 Agent 导出 + 会话导出的方式在设备间迁移数据：

1. 在旧设备上导出需要迁移的 Agent 和会话
2. 将导出的文件传输到新设备
3. 在新设备上分别导入
4. 重新关联 Agent 使用的服务渠道和模型

### 注意事项

- 导出的 Agent 不包含 API Key 等敏感信息
- 导入后需要重新选择可用的模型（如果原模型在目标设备上不存在）
- 资产文件在导出为 JSON 时不包含实际文件内容，需使用 ZIP 模式
