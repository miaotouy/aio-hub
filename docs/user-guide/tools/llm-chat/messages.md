# 💬 消息操作

LLM Chat 的消息系统基于**树状分支结构**——每条消息都可以产生多个回复，形成一棵对话树。这让你可以自由探索不同的对话方向，而不会丢失已有的内容。

## 发送消息

1. 在底部输入框中输入文本
2. 按 `Ctrl + Enter`（默认快捷键）发送
3. 消息会追加到当前会话的活动分支末尾

::: tip
输入框支持多行文本。要换行直接按 `Enter`，发送请使用 `Ctrl + Enter`。你可以在设置中交换这两个快捷键。
:::

### 输入工具栏

输入框上方提供了便捷工具：

- 📎 **附件按钮**：添加图片、文件等附件
- 🤖 **快速 Agent 切换**：在不离开对话框的情况下切换使用的 Agent
- ⚡ **快捷动作**：执行预设的快速操作模板
- 🔍 **上下文分析**：预览即将发送给 LLM 的完整上下文

## 消息列表操作

将鼠标悬停在消息上时，消息底部会显示**悬浮操作栏**，提供以下快捷操作：

### 主要操作（直接显示）

| 操作               | 说明                                                        |
| ------------------ | ----------------------------------------------------------- |
| 📋 **复制**        | 复制消息原文内容到剪贴板                                    |
| ✏️ **编辑**        | 修改已发送的消息内容                                        |
| 🔗 **创建分支**    | 在当前节点基础上开启一个新的平行对话路径                    |
| **重新生成**       | 让 AI 重新生成此回复                                        |
| @ **切换模型重试** | 选择其他模型后重新生成                                      |
| 👁️ **启用/禁用**   | 切换该消息是否参与上下文构建                                |
| 🗑️ **删除**        | 删除当前消息及其所有子分支                                  |
| 🌐 **翻译**        | 翻译消息内容，支持双语对照显示（按住 Shift 点击可快速翻译） |
| ⏹️ **终止**        | 生成过程中点击停止生成                                      |

### 更多菜单（`···` 按钮）

点击更多按钮可展开：

| 操作                   | 说明                             |
| ---------------------- | -------------------------------- |
| ⏩ **续写消息**        | 让 AI 从当前内容的末尾继续写下去 |
| @ **选择模型续写**     | 选择其他模型后续写               |
| 🛠️ **重新解析工具**    | 强制重新识别消息中的工具调用指令 |
| 📊 **上下文分析**      | 分析该消息在上下文中的位置和作用 |
| 📥 **导出分支**        | 导出该消息所在分支               |
| 🔢 **重新计算 Token**  | 重新计算本条消息的 Token 数      |
| 🗄️ **数据编辑 (高级)** | 直接编辑消息的原始数据           |

### 编辑与分支策略

点击 ✏️ 编辑按钮后，你可以：

- **保存修改**：直接覆盖当前节点（仅限尚未产生子节点的末梢消息）。
- **保存并创建新分支**：将修改后的内容作为一条新的平行消息发送，并基于此生成回复。这是探索“如果换一种说法”的最佳方式。

编辑已发送的用户消息时，系统默认会采用分支策略，以确保你不会丢失原本的对话路径。

## 分支管理

分支是 LLM Chat 最强大的特性之一。你可以从任意消息节点创建新的分支，探索不同的对话方向。

### 创建分支

- **从当前消息创建**：点击消息菜单的 🔗 按钮
- **自动分支**：如果当前节点已有子节点，发送新消息会自动创建一个新的分支

### 切换分支

- **分支选择器**：在消息上方或下方显示的分支标签，点击即可切换到其他分支
- **对话树图视图**：切换到「树图视图」可以直观地看到所有分支的完整结构

### 查看分支历史

`MessageNavigator` 组件提供了便捷的分支导航功能：

- **前进/后退**：在消息历史中导航
- **分支列表**：显示当前节点的所有分支，方便快速跳转

## 消息状态说明

| 状态   | 图标 | 含义                           |
| ------ | ---- | ------------------------------ |
| 完成   | ✅   | 消息已成功生成                 |
| 生成中 | ⏳   | AI 正在生成中                  |
| 错误   | ❌   | 生成过程出现错误               |
| 已禁用 | ⚪   | 该消息被用户禁用，不参与上下文 |

## 撤销与重做

- `Ctrl + Z`：撤销上一次的操作（如删除、编辑、分支切换）。
- `Ctrl + Shift + Z`：重做被撤销的操作。

撤销历史基于当前会话的 **全量快照 (Snapshot)** 机制维护——每次操作都会记录完整的消息树状态，确保复杂的消息树操作也能完美回滚。

## 性能与体验：原生虚拟渲染

为了处理包含数千条消息的长对话，LLM Chat 采用了 **CSS 原生虚拟渲染** 技术：

- **按需渲染**：只有位于视口内的消息才会被浏览器真正渲染，屏外消息仅保留布局占位。
- **零卡顿滚动**：相比传统的 JavaScript 虚拟滚动，原生技术提供了更平滑的滚动惯性和更好的可访问性。
- **位置保持**：当你切换分支或重新进入会话时，系统会自动恢复你上次浏览的确切位置，避免视觉跳动。
