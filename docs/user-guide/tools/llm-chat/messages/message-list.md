# 消息列表与悬浮操作栏

消息列表是对话的主战场。系统为每条消息提供了丰富的交互功能，通过悬浮操作栏即可快速触发。

## 1. 悬浮操作栏 (Message Toolbar)

将鼠标悬停在任意消息卡片上，底部会弹出操作栏：

### 常用操作

- **📋 复制**: 复制消息原文（Markdown 或纯文本）。
- **✏️ 编辑**: 进入编辑模式。
- **🔗 创建分支**: 在当前节点开启平行对话路径。
- **🔄 重新生成**: 让 AI 重新生成此条回复（可快速切换模型重试）。
- **👁️ 启用/禁用**: 切换该消息是否参与上下文组装。禁用的消息会变灰，且不会发送给 LLM。
- **🌐 翻译**: 实时翻译消息内容。按住 `Shift` 点击可快速触发默认翻译。

### 更多菜单 (`···`)

- **⏩ 续写**: 让 AI 从当前内容的末尾继续生成。
- **📊 上下文分析**: 查看该特定消息在当前上下文管道中的处理状态。
- **📥 导出**: 导出该消息所在的分支。
- **🔢 重新计算 Token**: 强制更新该消息的 Token 统计。
- **🗄️ 数据编辑**: 进入[高级数据编辑器](./message-data-editor)。

## 2. 编辑模式与分支策略

当你点击 ✏️ 编辑按钮时，系统提供两种保存方式：

1. **保存修改 (Overwrite)**: 直接覆盖当前节点。
   - _适用场景_: 修正错别字，或修改尚未产生子节点的最新消息。
2. **保存并创建新分支 (Branch)**:
   - _适用场景_: 修改已发送的消息，并希望基于新内容开启一段新的对话，同时保留旧的对话路径。
   - _默认行为_: 编辑已发送的用户消息时，系统通常会自动采用分支策略。

## 3. 原生虚拟渲染技术

为了确保在包含数千条消息的长对话中依然保持丝滑滚动，系统采用了**原生虚拟渲染**：

- **按需渲染**: 仅渲染视口内的消息，大幅降低内存占用。
- **位置保持**: 切换分支或重新进入会话时，系统会自动定位到你上次浏览的精确行。
