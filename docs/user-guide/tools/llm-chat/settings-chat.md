# ⚙️ 聊天参数设置

聊天参数控制 LLM 的行为方式和输出质量。参数配置有以下两个入口：

1. **聊天界面左侧栏**：切换到底部的「⚙️ 参数」标签页，即可调整当前 Agent 的模型参数。
2. **智能体编辑器**：在编辑 Agent 时，进入「角色设定」分区找到「模型」配置项，可以进行更详细的参数微调。

## 核心参数

### 温度 (Temperature)

控制输出结果的随机性和创造性：

- **范围**：0.0 ~ 2.0（具体取决于模型）
- **较低值 (0.1~0.3)**：输出更确定、更聚焦，适合翻译、事实性问答
- **中等值 (0.7~1.0)**：在创造性和连贯性之间取得平衡，适合通用对话
- **较高值 (1.0+)**：输出更多样、更有创意，但也可能更发散，适合头脑风暴

::: tip
Temperature 和 Top-P 通常不建议同时大幅度调整。一般固定一个，微调另一个。
:::

### Top-P（核采样）

控制模型从概率累积和达到 `p` 的候选词中采样：

- **范围**：0.0 ~ 1.0
- **较低值 (0.1~0.3)**：只考虑概率最高的少数词，输出更保守
- **较高值 (0.9~1.0)**：考虑更多候选词，输出更多样
- **建议**：配合 Temperature 使用，一般固定 Temperature 后微调 Top-P

### 最大 Token (Max Tokens)

控制单次生成的最大输出长度：

- **范围**：1 ~ 模型上下文窗口上限
- **较短 (512~1024)**：适合简短回复、分类任务
- **中等 (2048~4096)**：适合一般对话
- **较长 (4096+)**：适合长文生成、代码生成、详细分析
- **注意**：设置过大会增加响应延迟和 Token 消耗

## 高级参数

### 频率惩罚 (Frequency Penalty)

根据词在文本中出现的频率进行惩罚，减少重复：

- **范围**：-2.0 ~ 2.0
- **正值**：减少重复用词，让输出更多样
- **负值**：鼓励重复模式

### 存在惩罚 (Presence Penalty)

根据词是否已在文本中出现过进行惩罚，鼓励引入新话题：

- **范围**：-2.0 ~ 2.0
- **正值**：鼓励模型谈论新话题
- **负值**：鼓励模型在已有话题上深入

### Top-K

从概率最高的 K 个词中采样（部分模型支持）：

- **范围**：1 ~ 模型定义的上限
- **较低值**：输出更保守
- **较高值**：输出更多样

## 系统提示词 (System Prompt)

在智能体的「角色设定」部分，你可以编写系统提示词来定义 AI 的行为模式和角色。

### 编写建议

- **明确角色**：为 AI 设定清晰的身份，如「你是一个专业的 Python 程序员」
- **行为约束**：指定输出格式、语气、长度等约束
- **背景信息**：提供必要的上下文知识
- **使用分隔符**：在复杂提示中使用 Markdown 标题或 XML 标签组织内容

### 示例

```markdown
## 核心定位

你是一位经验丰富的技术写作专家。

## 行为准则

1. 回答应简洁、准确、有实操性
2. 优先提供代码示例
3. 不确定时如实说明
```

## 模型选择

在参数面板顶部，你可以选择当前智能体使用的**服务渠道**和**模型**：

1. **渠道 (Profile)**：选择已配置的 AI 服务（如 OpenAI、Anthropic 等）
2. **模型 (Model)**：从该渠道可用的模型列表中选择

::: tip
模型选择器会显示模型的上下文长度和能力标签（如「视觉」「工具调用」等），方便你选择最合适的模型。
:::

## 更多设置

除了每个智能体独立的参数外，全局聊天设置（入口：对话界面右上角的 ⚙️ 按钮）还包括：

- **UI 偏好**：流式输出开关、消息字体大小、时间戳显示等
- **主题命名**：自动生成会话标题的配置
- **请求设置**：超时时间、重试次数、增量保存间隔等
- **快捷键**：发送/换行快捷键绑定
