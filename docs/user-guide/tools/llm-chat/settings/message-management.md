# <LIcon name="MessageSquareMore" size="28" /> 消息管理

消息管理模块允许你平衡操作效率与数据安全，通过定制二次确认逻辑来防止因手滑导致的不可逆损失，默认为开启提醒。

## 风险控制 (Confirmation)

为了确保你的对话资产安全，系统为以下破坏性操作提供了独立的可选确认开关：

- **删除单条消息**: 开启后，点击消息侧边的 **“删除”** 按钮将弹出确认框。
- **删除整个会话**: 开启后，在会话列表或侧边栏点击 **“删除会话”** 时会要求确认。
- **清空所有记录**: 这是一个高危操作。开启后，执行“清空所有会话”时将强制进行二次确认。

---

::: warning 建议
**宁稳勿快**: 除非你正在进行大量的测试数据清理，否则我们强烈建议保持所有确认开关处于 **开启** 状态。LLM 生成的内容往往具有随机性，一旦误删，可能很难通过完全相同的 Prompt 再次找回。
:::
