# 全局正则替换规则 (Global Regex)

正则替换规则允许你在消息发送前自动对文本进行清洗、转换或增强。全局规则将应用于所有 Agent 和所有会话。

## 1. 使用场景

- **格式清理**: 自动移除冗余空行、特殊不可见字符或控制符。
- **隐私保护**: 自动模糊化或替换敏感词（如手机号、邮箱）。
- **排版标准化**: 自动在中文与英文/数字之间添加空格，统一全角/半角标点。
- **角色扮演增强**: 全局性的语气词转换或人称代词标准化。

## 2. 配置方式

在 **聊天设置 -> 文本处理 -> 全局文本替换** 中管理：

- **名称**: 规则的描述性名称（如“清理多余换行”）。
- **匹配模式**: 正则表达式。支持标准 JS 正则语法。
- **替换文本**: 替换后的内容。支持使用 `$1`, `$2` 引用捕获组。
- **启用状态**: 每个规则可独立开启或关闭。

## 3. 绑定模式 (Binding Mode)

系统提供了两种规则应用策略：

- **跟随消息配置 (Message-based)**: 
  - 消息在发送时，会记录下当时生效的正则规则。
  - 即使后续修改了全局规则，历史消息在重新生成或分析时仍使用其原始规则。
  - *优点*: 保证历史对话的一致性。
- **使用当前会话配置 (Session-based)**:
  - 所有消息（包括历史消息）统一应用当前最新的全局和 Agent 正则规则。
  - *优点*: 方便快速调整并看到全局效果。

## 4. 规则优先级

最终应用到文本的规则顺序为：
1. **全局规则**: 最先执行，处理通用格式。
2. **Agent 规则**: 其次执行，处理特定角色相关的转换。详见 [Agent 编辑器](../agents/editor-guide#4-输出与显示-output)。
