# 渲染与显示设置

控制消息内容在界面上的呈现方式，包括 Markdown 渲染和高级视觉效果。

## 1. 消息显示 (Display)

- **字体大小**: 调整消息文本的基础字号（支持 `rem` 单位以适配系统缩放）。
- **最大宽度**: 设置消息卡片在视口中占据的最大宽度比例。
- **头像显示**: 控制是否在消息旁显示用户和 Agent 的头像。

## 2. Markdown 渲染

系统内置了高性能的 Markdown 解析器，支持：
- **代码高亮**: 自动识别并渲染多种编程语言的代码块。
- **数学公式**: 支持 LaTeX 公式渲染。
- **表格与媒体**: 渲染标准的 Markdown 表格、图片和视频。
- **Mermaid 图表**: 开启后，支持直接在消息中渲染流程图、甘特图等 Mermaid 语法。

## 3. 视觉化输出 (Visual Output)

这是 LLM Chat 的特色功能，允许模型输出受控的 HTML/CSS/JS 内容。
- **启用视觉化预览**: 开启后，如果模型输出了符合规范的 HTML 代码块，系统会提供一个实时的预览窗口（Canvas）。
- **安全沙箱**: 所有的视觉化输出均运行在隔离的沙箱环境中，确保应用安全。
- **主题同步**: 配合 `{{cssvar}}` 宏，视觉化输出可以自动适配当前的明暗主题。

## 4. 思考块显示 (Thought Blocks)

针对具有推理能力的消息（如 DeepSeek-R1、OpenAI o1），系统支持自定义思考过程的显示：
- **显示模式**: 折叠显示、展开显示或完全隐藏。
- **识别规则**: 配置识别思考过程的标签（如 `<think>`）。
