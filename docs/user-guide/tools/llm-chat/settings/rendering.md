# <LIcon name="Zap" size="28" /> 渲染设置

渲染设置模块决定了聊天消息如何被解析、展示以及在流式生成过程中的性能表现。

## 核心引擎 (Core Engines)

- **消息渲染器**:
  - 选择 Markdown 解析引擎版本。不同版本在解析速度和对复杂语法（如数学公式、图表）的支持上有所差异。
- **代码块高亮**:
  - 消息代码块固定使用 CodeMirror，以降低启动和长会话渲染成本，并适配流式打字机效果。

## 性能与平滑度 (Performance & Smoothing)

- **渲染节流 (Throttling)**:
  - **说明**: 控制流式输出时 AST 树的更新频率。
  - **作用**: 设置为 100ms 左右可显著降低生成超长文本时的 CPU 负载，防止界面假死。
- **流式平滑化 (Streaming Smoothing)**:
  - **功能**: 通过语义分块技术，消除流式输出时常见的“文字跳跃”感，使打字机效果如丝般顺滑。
- **HTML 预览冻结 (Freezer)**:
  - **核心机制**: 当会话中出现多个 HTML 预览（如连续生成多个网页）时，自动冻结非最新的预览容器。
  - **优势**: 极大降低了 CPU 和内存开销，同时保持最近 N 条消息的预览依然可交互。

## 安全与护栏 (Safety Guard)

- **渲染安全护栏**:
  - **说明**: 自动检测并限制异常节点数、深度嵌套及解析超时。
  - **作用**: 防止模型因输出恶意构造的特殊字符或无限循环内容而导致前端崩溃。
- **HTML 安全策略**:
  - **CDN 本地化**: 拦截并重定向常用 CDN 资源至本地，提升加载速度并支持离线预览。
  - **危险标签限制**: 严格过滤 `script`、`iframe` 等标签，除非你手动开启“允许危险标签”开关。

## 交互特性

- **性能指标 (Metrics)**: 开启后可在消息底部看到实时的生成速度 (tokens/s) 及首字响应延迟 (TTFT)。
- **无缝模式 (Seamless Mode)**: 移除 HTML 预览的外框，使其看起来像原生 UI 一样嵌入在消息流中。

---

::: warning 注意
开启“允许危险 HTML 标签”可能会让 AI 生成的内容执行恶意脚本。请仅在调试或完全信任模型输出的情况下使用。
:::
