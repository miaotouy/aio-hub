# <LIcon name="PenTool" size="28" /> 话题命名

话题命名模块负责将会话从枯燥的“新会话”自动转换为具有描述性的标题，方便你在侧边栏快速定位历史记录。

## 自动化策略 (Automation)

- **启用话题命名**: 总开关。开启后，系统将监控对话进度。
- **自动触发阈值**:
  - **说明**: 设置在用户发送多少条消息后触发自动命名。
  - **建议**: 设为 2-3 条。此时对话主题通常已经明确，生成的标题最为准确。
- **上下文引用数**:
  - **说明**: 生成标题时参考的最近消息条数。
  - **作用**: 确保模型能捕捉到最新的讨论核心。

## 模型与指令 (Model & Prompt)

- **命名模型**:
  - **建议**: 使用轻量级模型（如 GPT-4o-mini 或本地小型模型）。命名任务对逻辑要求不高，小模型速度更快且成本更低。
- **命名提示词 (Prompt Editor)**:
  - **高度灵活**: 你可以要求模型生成特定风格的标题（如：极其简短、包含 Emoji、学术风格）。
  - **占位符**: 使用 `{context}` 标记对话内容注入的位置。如果未提供，内容将自动追加到提示词末尾。

## 生成控制

- **温度 (Temperature)**: 建议设为 0.1 - 0.3。较低的随机性有助于生成更直白、不花哨的标题。
- **输出上限 (Max Tokens)**: 建议限制在 20-50 tokens 以内，防止模型生成一段话作为标题。

---

::: tip 提示
**手动干预**: 即使自动生成了标题，你随时可以双击侧边栏的会话项或在会话设置中手动修改标题。手动修改后的标题不会被自动命名再次覆盖。
:::
