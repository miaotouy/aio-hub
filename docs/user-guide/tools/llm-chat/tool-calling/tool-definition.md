# 工具定义与宏注入

智能体（Agent）感知工具能力的核心在于提示词（Prompt）中的工具定义。系统利用宏引擎实现了动态、按需的工具注入。

## 关键宏定义

在智能体的“角色设定”或“预设消息”中，可以使用以下宏：

-   **`{{tools}}`**：
    -   **功能**：全量注入。根据当前智能体勾选启用的工具列表，生成完整的描述和参数 Schema。
    -   **保底机制**：如果配置中启用了工具调用但提示词中缺失此宏，系统可配置“保底注入”，自动将定义插入到对话历史之前。
-   **`{{tool_usage}}`**：
    -   **功能**：注入协议说明。告诉模型如何构造调用指令（例如 VCP 协议的具体语法格式）。
-   **`{{tools::toolId}}`**：
    -   **功能**：精确注入。仅注入指定 ID 的工具定义，适用于需要精简上下文的场景。

## 方法级深度配置

在“可用工具列表”中展开具体工具，可以进行微观控制：

1.  **方法开关**：控制模型是否能看到并调用该工具下的特定子功能。
2.  **描述覆盖 (Overrides)**：
    -   你可以修改工具或方法的**显示名称**和**描述**。
    -   **用途**：如果默认的工具描述（通常由开发者编写）不够直观，或者你希望通过特定的引导词让模型更准确地理解何时使用该工具，可以通过覆盖功能进行调优。
3.  **示例注入**：为方法提供具体的调用示例，通过 Few-shot 增强模型的遵循能力。

## 配置迁移

-   **复制/粘贴 JSON**：工具面板支持一键复制所有工具的启用状态和覆盖配置。
-   **用途**：方便在不同的智能体之间快速同步复杂的工具调用策略，或备份调优后的描述。
