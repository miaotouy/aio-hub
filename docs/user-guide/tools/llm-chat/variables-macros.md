# 🔣 变量与宏

<div v-pre>

LLM Chat 提供了一套强大的**宏系统**和**会话变量系统**，让你能够在提示词中嵌入动态内容，实现高度灵活的对话配置。

## 宏系统 (Macro System)

宏是一种用 `{{ }}` 包裹的占位符，在处理上下文时会被自动展开为实际内容。

### 基础语法

<div v-pre>


`{{macroName}}` 无参数宏
`{{macroName::arg1::arg2}}` 带参数宏
`\{{example}}` 转义：输出原文字符，不触发宏处理


</div>

### 转义规则

如果你需要输出 `{{` 字面量而不触发宏，在前面加反斜杠：`\{{text}}` → `{{text}}`

### 空白符控制

`{{trim}}` 是一个特殊标记，它不输出内容，但会移除其前后的所有空白字符（包括换行符）。

## 核心上下文宏

| 宏                       | 说明                         |
| ------------------------ | ---------------------------- |
| `{{user}}`               | 当前用户档案的名称           |
| `{{char}}` / `{{agent}}` | 当前 Agent 的名称            |
| `{{persona}}`            | 当前用户档案的完整内容       |
| `{{description}}`        | Agent 的描述信息             |
| `{{personality}}`        | Agent 的性格特征（尚未实现） |
| `{{scenario}}`           | 当前对话场景（尚未实现）     |
| `{{input}}`              | 当前输入内容（或选中的内容） |
| `{{lastMessage}}`        | 会话中的最后一条消息         |
| `{{lastUserMessage}}`    | 最后一条用户消息             |
| `{{lastCharMessage}}`    | 最后一条助手消息             |

| `{{date_ymd}}` | `2026-05-08`          |
| `{{datetime}}` | `2026-05-08 22:30:00` |

## 变量操作宏

变量系统有两种生命周期：

### 局部变量 (单次请求内有效)

| 宏                        | 说明         | 示例                      |
| ------------------------- | ------------ | ------------------------- |
| `{{setvar::name::value}}` | 设置临时变量 | `{{setvar::mood::happy}}` |
| `{{getvar::name}}`        | 读取临时变量 | `{{getvar::mood}}`        |
| `{{incvar::name}}`        | 变量自增 1   | `{{incvar::counter}}`     |
| `{{decvar::name}}`        | 变量自减 1   | `{{decvar::counter}}`     |

::: tip
局部变量在单次 LLM 请求的处理过程中持续存在，按预设消息的物理顺序从上到下串行执行。它们不会跨轮次持久化。
:::

### 全局变量 (应用进程内有效)

| 宏                           | 说明           | 示例                            |
| ---------------------------- | -------------- | ------------------------------- |
| `{{setglobalvar::key::val}}` | 设置全局变量   | `{{setglobalvar::mood::happy}}` |
| `{{getglobalvar::key}}`      | 读取全局变量   | `{{getglobalvar::mood}}`        |
| `{{incglobalvar::name}}`     | 全局变量自增 1 | `{{incglobalvar::counter}}`     |
| `{{decglobalvar::name}}`     | 全局变量自减 1 | `{{decglobalvar::counter}}`     |

全局变量在整个应用运行期间有效，可以跨会话跨轮次持久化，直到应用关闭。

## 功能性宏

| 宏                        | 说明                     | 示例                           |
| ------------------------- | ------------------------ | ------------------------------ |
| `{{random::opt1::opt2}}`  | 随机选择                 | `{{random::高冷::热情::神秘}}` |
| `{{randomInt::min::max}}` | 生成指定范围内的随机整数 | `{{randomInt::1::10}}`         |
| `{{pick::A::B}}`          | 基于会话内容的稳定选择   | 同一会话结果固定               |
| `{{roll::NdM}}`           | 掷骰子                   | `{{roll::1d100}}`              |
| `{{repeat::text::n}}`     | 重复文本                 | `{{repeat::*::5}}`             |
| `{{newline}}`             | 插入换行符               | `第一行{{newline}}第二行`      |

| 宏                            | 说明                                                    |
| ----------------------------- | ------------------------------------------------------- |
| `{{assets::groupId}}`         | 列出 Agent 的可用资产。`groupId` 可选                   |
| `{{tools::toolId1::toolId2}}` | 注入工具定义。支持参数指定工具 ID 列表（无视启用开关）  |
| `{{tool_usage}}`              | 注入工具调用协议的使用说明（如 VCP 格式说明）           |

| `{{cssvar::--var-name}}`      | 获取 CSS 变量的当前实际值（用于视觉化输出）             |

## 宏的执行机制

宏引擎采用三阶段管道执行：

1. **预处理 (PRE_PROCESS)**：处理状态变更宏（如 `setvar`）
2. **替换 (SUBSTITUTE)**：处理静态值替换（如 `user`, `getvar`）
3. **后处理 (POST_PROCESS)**：执行动态计算（如 `time`, `lastMessage`）

这确保了宏之间的依赖关系被正确处理。

## 会话变量系统

除了宏系统中的临时变量外，LLM Chat 还支持持久化的**会话变量 (Session Variables)**：

- 你可以在智能体的 `Variable Config` 中定义变量树
- 支持嵌套分组和具体的变量定义
- 每个变量有初始值、最小值/最大值约束
- 变量的变化历史会被记录在消息元数据中

### 配置方式

在智能体编辑器中找到「变量配置」部分：

1. 启用变量系统
2. 定义变量树（支持分组和子变量）
3. 设置每个变量的初始值、范围约束
4. 可选：自定义 UI 样式

### 运行机制

- 变量值在每次消息发送时生成快照
- 快照存储在消息的 `metadata.sessionVariableSnapshot` 中
- 你可以在上下文分析器中查看变量状态和变化历史

## 最佳实践

1. **动态角色设定**：在系统提示词中使用 `{{time}}` 和 `{{date}}` 让角色感知时间
2. **通用清洗规则**：在正则替换中使用 `{{user}}` 编写不依赖具体用户名的规则
3. **跨预设传递状态**：在靠前的预设消息中 `setvar`，在后面的消息中 `getvar`
4. **随机化开场白**：使用 `{{random::你好::嗨::哟}}` 让 AI 的开场白不重复
