# 📖 世界书 (Worldbook)

世界书 (Worldbook) 是一种**关键词触发的上下文注入机制**，灵感来自 SillyTavern 的 World Info / Lorebook 功能。它允许你预先定义大量背景设定，然后在对话中通过关键词自动匹配并注入相关内容。

## 世界书的概念

想象你正在写一部奇幻小说——你需要记录大量设定：人物关系、地理环境、魔法体系、历史事件……世界书就是这些设定的数据库。

与传统的一整段系统提示词不同，世界书采用**"按需注入"**策略：只有当前对话中出现了相关关键词，对应的设定才会被加入上下文。这让你可以在不撑爆 Token 预算的前提下，维护一个庞大的知识库。

## 创建世界书

### 从零创建

1. 打开聊天设置 → 世界书 → 点击「世界书库管理」
2. 点击「新建世界书」
3. 输入名称，系统会自动创建一个空白世界书

### 从 SillyTavern 导入

支持导入 SillyTavern 格式的 World Info JSON 文件：

1. 在世界书管理器中点击「导入」
2. 选择 `.json` 文件
3. 系统会自动解析所有条目和元数据

## 管理条目

每个世界书包含多个条目 (Entry)。在条目编辑器中你可以配置：

### 基本信息

| 字段               | 说明                   |
| ------------------ | ---------------------- |
| **UID**            | 条目的唯一编号         |
| **备注 (Comment)** | 条目的描述或说明       |
| **内容 (Content)** | 条目被触发时注入的文本 |

### 触发规则

关键词匹配触发：

- **主要关键词 (Key)**：条目的主要触发词
- **次要关键词 (Key Secondary)**：需要配合主要关键词的逻辑条件
- **大小写敏感**：可选，默认不区分大小写
- **全词匹配**：可选，只匹配完整单词
- **选择性逻辑**：AND_ANY / NOT_ALL / NOT_ANY / AND_ALL 等逻辑组合

### 常量条目

将 `constant` 设置为 `true` 后，该条目会在**每次请求中始终注入**，不依赖关键词匹配。适合存放长期有效的背景设定。

### 插入位置

| 位置                 | 说明                             |
| -------------------- | -------------------------------- |
| **Before Character** | 插入在所有角色定义之前           |
| **After Character**  | 插入在角色定义之后、对话历史之前 |
| **Before AN**        | 插入在作者备注之前               |
| **After AN**         | 插入在作者备注之后               |
| **Depth (深度)**     | 插入到对话历史的指定深度位置     |
| **Outlet**           | 插入到指定的输出点               |

### 高级控制

- **概率触发**：设置触发概率（0-100%）
- **粘性激活**：激活后持续 N 条消息
- **冷却时间**：激活后 N 条消息内不再激活
- **延迟激活**：前 N 条消息不激活
- **分组与组内权重**：同组条目发生竞争，设置权重分配
- **扫描深度**：回溯多少条消息进行关键词匹配
- **角色过滤**：仅对特定角色生效

## 关联世界书

### 全局关联

在聊天设置 → 世界书中，可以添加全局生效的世界书，应用于所有会话和 Agent。

### Agent 关联

在 Agent 编辑器中，可以为特定 Agent 绑定独立的世界书列表。

### 配置覆盖

Agent 可以覆盖全局的世界书扫描设置：

- **禁用递归扫描**：条目标内容不再触发递归匹配
- **默认扫描深度**：覆盖全局的默认深度设置

## 世界书与上下文管道

Worldbook Processor 在上下文管道中负责执行匹配和注入。它的工作流程：

1. 获取当前消息历史
2. 从历史中提取关键词（支持递归扫描）
3. 在关联的世界书中匹配条目
4. 按优先级、位置和顺序排序匹配结果
5. 注入到指定位置

你可以在上下文分析器的结构化视图中看到哪些世界书条目被匹配和注入。
