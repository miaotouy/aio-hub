# 世界书条目管理

世界书由多个条目 (Entry) 组成。每个条目都代表一段特定的背景知识或规则。

## 1. 条目编辑器

在世界书管理器中选择一个库，即可看到条目列表。点击条目可进入详细编辑器：

### 1.1 基本信息

- **UID**: 条目的唯一编号，方便内部引用。
- **备注 (Comment)**: 用于记录该条目的简要描述（如“主角的佩剑设定”），不会发送给 LLM。
- **内容 (Content)**: 条目被触发时实际注入到上下文中的文本。

### 1.2 触发规则

- **主要关键词 (Key)**: 触发该条目的核心词汇。支持多个词，用逗号分隔。
- **次要关键词 (Key Secondary)**: 可选。用于建立复杂的触发逻辑（如：只有当出现“剑”且同时出现“亚瑟”时才触发）。
- **匹配选项**:
  - **大小写敏感**: 是否区分英文大小写。
  - **全词匹配**: 是否只匹配独立的单词（防止“apple”匹配到“pineapple”）。
  - **逻辑组合**: AND_ANY / NOT_ALL / NOT_ANY / AND_ALL 等，控制主次关键词的逻辑关系。

### 1.3 常量条目 (Constant)

将 `constant` 设置为 `true` 后，该条目将**始终被注入**到上下文中，不再依赖关键词匹配。这适用于那些必须始终存在的全局设定。

## 2. 注入配置

你可以精确控制条目注入到提示词中的位置：

| 位置                 | 说明                                                    |
| :------------------- | :------------------------------------------------------ |
| **Before Character** | 插入在所有 Agent 角色定义之前。                         |
| **After Character**  | 插入在角色定义之后、对话历史之前。                      |
| **Before AN**        | 插入在作者备注 (Author's Note) 之前。                   |
| **After AN**         | 插入在作者备注之后。                                    |
| **Depth (深度)**     | 插入到对话历史的指定深度位置（例如倒数第 5 条消息处）。 |

## 3. 高级控制

- **概率触发**: 设置一个 0-100% 的概率，让条目随机出现，增加对话的不可预测性。
- **粘性激活 (Sticky)**: 触发一次后，强制在后续 N 条消息中保持激活状态。
- **冷却时间 (Cooldown)**: 激活后，在接下来的 N 条消息内禁止再次触发。
- **分组权重**: 将多个条目分配到同一个组（Group），并设置不同的权重，实现同类设定间的竞争或随机选择。
