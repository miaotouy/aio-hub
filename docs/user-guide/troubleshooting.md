# 故障排除指南

## 概述

使用 AIO Hub 过程中遇到问题？别着急。本指南汇总了最常见的故障类型和对应的解决方法，你可以根据自己的情况快速定位问题。

> 💡 **自助排查原则**：遇到问题先重启应用——很多临时问题重启就能解决。如果重启无效，再按本指南的步骤逐一排查。

---

## 安装问题

### Windows 安装时提示 "无法验证发布者"

**症状**：安装时 Windows 弹出安全警告，显示"Windows 保护了你的电脑"。

**原因**：AIO Hub 的安装包尚未完成微软的签名认证流程，这是首次安装未签名应用时的正常提示。

**解决步骤**：

1. 在安全警告弹窗中，点击 **"更多信息"**（蓝色文字链接）
2. 底部会出现 **"仍要运行"** 按钮，点击即可继续安装
3. 安装完成后不会再弹出此提示

> 如果你还是不放心，可以查看项目在 GitHub 上的开源代码，或对照 Releases 页面提供的 SHA-256 校验码验证文件完整性。

### macOS 安装后提示 "应用已损坏，无法打开"

**症状**：从官网下载的 `.dmg` 安装完成后，双击应用提示"应用已损坏，无法打开。你应该将它移到废纸篓"。

**原因**：macOS Gatekeeper 安全机制拦截了未经 Apple 公证的第三方应用。**这不是应用本身的问题**。

**解决步骤**：

**方法一（推荐）**：

1. 在 `应用程序` 文件夹中找到 AIO Hub
2. 按住 `Control` 键，同时右键点击应用图标
3. 选择 **"打开"**
4. 在弹出的确认弹窗中点击 **"打开"**
5. 之后就可以正常双击打开了

**方法二（如果方法一无效）**：

1. 打开 **终端（Terminal）**
2. 执行以下命令（需要输入开机密码）：
   ```bash
   sudo xattr -cr /Applications/AIO\ Hub.app
   ```
3. 完成后重新启动应用

> 更详细的说明请参考 [macOS Gatekeeper 修复指南](./faq/macos-gatekeeper-fix.md)。

### Linux 安装后无法启动（无反应）

**症状**：双击 AppImage 文件后没有任何反应。

**原因**：AppImage 文件没有执行权限。

**解决步骤**：

1. 在终端中执行：
   ```bash
   chmod +x AIO-Hub_*.AppImage
   ```
2. 或者右键文件 → **属性 → 权限**，勾选 **"允许以程序执行文件"**
3. 再双击运行即可

> 如果依然无法启动，尝试从终端运行查看错误输出：
>
> ```bash
> ./AIO-Hub_*.AppImage
> ```

## 运行问题

### 应用启动后白屏 / 界面空白

**症状**：应用能打开，但界面完全空白，没有任何内容。

**原因**：

- **系统组件缺失**：在 Windows 上可能表现为 WebView2 损坏或缺失（现代系统较少见）；在 Linux 上则更为常见，通常是由于缺少必要的 WebKit 依赖库。
- **系统显卡驱动过旧**：显卡驱动不支持硬件加速渲染。

**解决步骤**：

1. **Windows 用户**：检查 WebView2 状态。若系统较老或为精简版，可尝试手动安装：[下载 Microsoft Edge WebView2 Runtime](https://developer.microsoft.com/zh-cn/microsoft-edge/webview2/)。
2. **Linux 用户**：确保已安装 `webkit2gtk` 等必要的系统库。
3. **更新显卡驱动**：前往你的显卡厂商官网（NVIDIA / AMD / Intel）下载最新驱动
4. **尝试禁用硬件加速**：如果应用能进入设置页面，前往 **设置 → 外观**，关闭硬件加速

### 应用频繁闪退

**症状**：使用过程中应用突然关闭，没有任何提示。

**原因**：可能是内存不足、系统兼容问题或特定操作触发了异常。

**解决步骤**：

1. **重启应用**：先试最基础的方法
2. **查看系统资源**：打开任务管理器（Windows）或活动监视器（macOS），确认内存和 CPU 使用率是否过高
3. **清理缓存**：关闭应用后，手动删除缓存目录中的数据（注意：这不会删除你的配置和聊天记录）
   - Windows：`%APPDATA%\com.mty.aiohub\cache`
   - macOS：`~/Library/Caches/com.mty.aiohub`
   - Linux：`~/.cache/com.mty.aiohub`
4. **查看日志**：如果闪退频繁，查看日志文件中的错误信息：
   - Windows：打开 `%APPDATA%\com.mty.aiohub\logs\` 文件夹，找到最新的 `.log` 文件
   - macOS：打开 `~/Library/Logs/com.mty.aiohub/` 文件夹
   - Linux：打开 `~/.local/share/com.mty.aiohub/logs/` 文件夹
5. **反馈问题**：将日志文件中的相关错误信息截图或复制，到 [GitHub Issues](https://github.com/miaotouy/aio-hub/issues) 反馈，开发者可以快速定位问题

### 应用运行缓慢 / 卡顿

**症状**：界面操作响应慢，切换页面有明显延迟。

**原因**：

- 打开了过多工具或标签页
- 长时间运行未重启，内存累积占用过高
- 系统资源不足

**解决步骤**：

1. **关闭不用的工具页**：只保留当前正在使用的工具
2. **重启应用**：释放累积的内存占用
3. **检查后台任务**：如果有大量的队列任务或数据处理任务在后台运行，等待它们完成或手动取消
4. **降低界面特效**：前往 **设置 → 外观**，关闭毛玻璃效果和界面效果，可以减少 GPU 负担
5. **可能存在漏洞**: 收集相关信息到 **issues** 中反馈

### 开启毛玻璃效果后界面闪烁

**症状**：开启毛玻璃（透明模糊）效果后，窗口或部分区域闪烁、渲染异常。

**原因**：显卡驱动兼容性问题，或系统不支持该特效。

**解决步骤**：

1. 前往 **设置 → 外观**，将 **"界面质感"** 调整为 **"纯色"** 模式
2. 如果问题消失，说明是显卡驱动问题。尝试更新显卡驱动到最新版本
3. 如果更新驱动后问题依然存在，保持"纯色"模式使用，不影响核心功能

## 网络问题

### LLM 对话时提示 "连接超时" 或 "请求失败"

**症状**：在聊天或任何使用 AI 服务的工具中，请求一直转圈，最终提示连接超时。

**原因**：

- API 服务提供商在中国大陆访问不稳定（如 OpenAI）
- 代理配置不正确或系统的其他网络问题
- 上游服务存在问题，尝试切换服务商

**解决步骤**：

1. **检查代理配置**：
   - 如果你需要走代理访问 API，请在 **设置 → 通用设置 → 网络代理** 中配置 HTTP 代理
   - 注意代理地址的格式：`http://127.0.0.1:端口号`，不要漏掉 `http://`
2. **检查网络连接**：打开浏览器访问 https://www.baidu.com 确认网络正常
3. **尝试直连**：如果配置了代理，先关闭代理试试直连是否能正常工作
4. **更换模型服务商**：如果某个服务商稳定性差，可以尝试其他可用的服务商

### 提示 "SSL 证书错误"

**症状**：请求 API 时返回 SSL 相关的错误信息。

**原因**：

- 系统时间不准确
- 代理软件拦截了 SSL 证书
- 企业网络环境中使用了自签名证书

**解决步骤**：

1. **校准系统时间**：确保电脑的日期和时间是正确的
2. **检查代理**：如果使用了第三方代理软件，尝试暂时关闭或切换代理模式
3. **尝试更换网络**：切换到手机热点或其他网络，排除企业网络限制
4. **在设置中调整**：前往 **设置 → 网络**，查看是否有 SSL 验证相关的选项

### 图片或文件上传失败

**症状**：上传附件或图片时转圈后失败。

**原因**：

- 文件过大
- 网络上传速度慢
- 存储空间不足

**解决步骤**：

1. **检查文件大小**：尝试上传小文件（1MB 以下），如果成功说明是文件大小限制
2. **检查磁盘空间**：确认系统盘还有足够的剩余空间
3. **重启应用**：偶尔是临时网络波动，重试即可

## 数据问题

### 意外关闭后数据丢失

**症状**：应用意外退出后，重启发现部分对话记录或配置丢失。

**原因**：

- 数据尚未完成自动保存
- 配置文件损坏

**解决步骤**：

1. **检查自动保存**：AIO Hub 会在操作过程中自动保存数据。如果只是个别最近的对话丢失，属于正常现象
2. **查看日志文件**：在日志目录中检查是否有 `storage` 相关的错误信息
3. **备份恢复**：如果你之前手动备份过数据，可以将备份文件复制回数据目录覆盖

**数据目录位置**：
| 平台 | 路径 |
| :--- | :--- |
| **Windows** | `%APPDATA%\com.mty.aiohub` |
| **macOS** | `~/Library/Application Support/com.mty.aiohub` |
| **Linux** | `~/.local/share/com.mty.aiohub` |

### 如何手动备份数据

**建议定期备份**，特别是当你有重要的对话记录或配置时。

**手动备份步骤**：

1. 关闭 AIO Hub（确保数据文件没有被占用）
2. 找到你的数据目录（见上表）
3. 复制整个目录到安全位置（如移动硬盘、云盘）
4. **恢复时**：同样先关闭应用，将备份的目录覆盖回原位置即可

> 📅 **建议频率**：如果你重度使用 AI 对话，建议每周备份一次。只在本地使用的话，每月备份一次已经足够。

### 配置修改后恢复默认值

**症状**：修改了某个设置后，应用行为异常，或者想回到最初的状态。

**解决步骤**：

1. 关闭 AIO Hub
2. 找到数据目录中的配置文件（通常是 `config.json` 或 `settings.json`）
3. 将该文件重命名（例如改为 `config.json.bak`）
4. 重新启动应用——它会自动生成一份新的默认配置文件
5. 如果问题解决，你可以手动对照备份文件，只迁移重要的设置项

> ⚠️ 注意：此操作会丢失你的所有设置，建议先完整备份整个数据目录。

## 常见问题 (FAQ)

### Q1：在哪里可以找到应用的错误日志？

日志文件是排查问题的关键。按你的平台找到对应路径：

| 平台        | 日志路径                              |
| :---------- | :------------------------------------ |
| **Windows** | `%APPDATA%\com.mty.aiohub\logs\`      |
| **macOS**   | `~/Library/Logs/com.mty.aiohub/`      |
| **Linux**   | `~/.local/share/com.mty.aiohub/logs/` |

日志文件按日期命名（如 `app-2025-05-07.log`），最新的问题可以在最新日期的文件中找到。如果你需要向开发者反馈问题，请附上相关日志内容。

### Q2：如何确认我下载的版本是最新的？

应用启动时会自动检查更新。你也可以手动检查：

- 进入 **设置 → 关于**，点击 **"检查更新"**
- 或者直接访问 [GitHub Releases 页面](https://github.com/miaotouy/aio-hub/releases)
- 推荐使用 **Pre-release（预览版）** 第一时间体验新功能

### Q3：代理已经配置了，但还是连不上 API？

按以下顺序检查：

1. **确认代理地址格式正确**：应该像 `http://127.0.0.1:7897` 这样，不要漏掉 `http://`
2. **确认代理服务正在运行**：打开代理软件的界面，确认它正常工作
3. **测试代理连通性**：在终端执行 `curl -x http://127.0.0.1:7897 https://api.openai.com/v1/models`（Windows 用户可以使用 PowerShell 的 `Invoke-WebRequest` 命令）
4. **尝试切换代理模式**：有些代理软件需要切换到"全局模式"才能被非浏览器应用使用

### Q4：切换主题后界面显示异常？

**症状**：从暗色主题切换到亮色主题（或反之）后，部分区域显示不正常。

**解决步骤**：

1. 尝试 **重启应用**——大部分主题切换问题重启即可修复
2. 如果问题持续，前往 **设置 → 外观**，手动切换一次主题模式
3. 如果依然异常，尝试清除浏览器缓存（应用内缓存，不影响数据）：找到缓存目录并删除其中的内容
   - Windows：`%APPDATA%\com.mty.aiohub\cache`
   - macOS：`~/Library/Caches/com.mty.aiohub`
   - Linux：`~/.cache/com.mty.aiohub`

### Q5：聊天记录中有敏感信息，如何彻底清除？

1. 在聊天工具中，找到对应的对话，点击删除即可
2. 如果需要批量清除，可以在聊天管理页面选择多条对话进行删除
3. 如果你希望彻底清除所有数据（包括配置），关闭应用后删除整个数据目录（见上文"数据目录位置"），重启应用即可
   > ⚠️ 此操作不可逆，请先确认你不需要保留任何数据

### Q6：用着用着界面变成了英文？

**症状**：应用界面语言突然从中文变成了英文。

**原因**：通常是自动检测系统语言时发生了切换，或者误触了语言设置。

**解决步骤**：

1. 前往 **设置 → 外观 → 语言**
2. 选择 **"简体中文"**
3. 应用会自动切换回中文界面
4. 如果还是没有变化，重启应用即可

### Q7：如何在旧版本之间迁移数据？

如果你从非常早期的版本（v0.3.x 或更早）升级到最新版：

1. 先找到旧版本的数据目录并完整备份
2. 安装新版本，首次运行后关闭应用
3. 将备份的数据目录中的 `config.json` 和 `chat-history/` 等关键目录复制到新版本的数据目录中
4. 启动应用，检查配置和聊天记录是否正常迁移

> 从 v0.4.x 版本开始，所有版本之间自动兼容，覆盖安装即可。

### Q8：应用卡住了，无法正常关闭怎么办？

**Windows 用户**：

1. 按 `Ctrl + Shift + Esc` 打开任务管理器
2. 找到 **AIO Hub**，右键 → **结束任务**
3. 重新启动应用

**macOS 用户**：

1. 点击顶部菜单栏的苹果图标 → **强制退出**
2. 选择 **AIO Hub**，点击 **"强制退出"**

**通用方法**：

- 按 `Alt + F4`（Windows）或 `Cmd + Q`（macOS）强制退出
- 如果仍然无法关闭，重启电脑是最简单的解决方案
