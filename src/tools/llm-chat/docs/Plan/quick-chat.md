# 快速会话（无痕模式）实现方案

> **文档状态 (Status)**: `Draft`
> **版本 (Version)**: 1.0
> **日期 (Date)**: 2026-05-09
> **负责模块**: `llm-chat`

---

## 1. 背景与目标

### 1.1 背景

目前所有的聊天会话都会自动持久化到磁盘中。当用户只是想进行一次性的模型测试、简单的即兴提问或不希望留下记录的对话时，现有的“强制保存”机制显得过于沉重。

### 1.2 目标

1. **无痕体验**：提供一种“快速会话”模式，其内容仅存在于内存中，应用重启或手动清空后即刻消失，不占用磁盘空间。
2. **常驻入口**：在会话列表顶部提供一个固定的“快速会话”项，方便用户随时发起对话。
3. **按需持久化**：支持将临时的快速会话内容一键保存为正式的持久化记录。
4. **与智能体解耦**：快速会话既可以使用选中的智能体，也可以在“纯净模式”下运行（依赖 Agent-Session 解耦方案）。

---

## 2. 技术方案

### 2.1 数据结构增强 (`types/session.ts`)

在会话索引中增加 `isTemporary` 标记，用于在存储层识别是否需要跳过持久化。

```typescript
export interface ChatSessionIndex {
  // ...
  /** 临时会话标记：仅存于内存，不写入索引文件 */
  isTemporary?: boolean;
}
```

### 2.2 存储层拦截 (`useChatStorageSeparated.ts`)

修改持久化逻辑，在执行保存操作前过滤掉带有 `isTemporary` 标记的会话。

```typescript
// 过滤掉临时会话，确保它们不会被写入磁盘
const persistentIndexes = sessions.filter((s) => !s.index.isTemporary);
```

### 2.3 初始化与常驻逻辑 (`llmChatStore.ts`)

在应用启动加载会话列表时，手动在内存中创建一个 ID 为 `session-quick-chat` 的固定会话。

- **ID**：`session-quick-chat`（常量）。
- **状态**：默认开启，不可删除，不可通过拖拽排序。

### 2.4 UI 适配

#### 会话侧边栏 (`SessionsSidebar.vue`)

- 在列表最上方固定渲染“快速会话”项。
- **视觉**：使用特殊的图标（如 `Zap`）以区别于普通会话。
- **交互**：点击直接进入该内存会话。

#### 聊天区域 (`ChatArea.vue`)

- **保存功能**：在快速会话激活时，工具栏或头部显示“保存为正式记录”按钮。点击后，清除 `isTemporary` 标记并执行一次全量持久化。
- **清空功能**：提供快捷按钮一键重置该内存会话的所有节点。

## 4. 注意事项

- **内存管理**：虽然不占磁盘，但大量的快速会话消息仍会占用内存。建议在用户切换工具或长时间未活动时，提供自动清空或提醒机制。
- **状态同步**：如果用户在分离窗口中操作快速会话，需要确保主窗口的内存状态能实时同步。
