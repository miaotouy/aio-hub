## 2. 已完成功能清单

### 2.1 核心翻译能力

- 多渠道并发翻译，每个渠道独立 `AbortController`，互不阻塞
- 渠道粒度的流式输出（设置可关）
- 渠道粒度的"停止 / 重试"按钮
- 流式累积内容与最终响应的"最长保护"，规避不同 LLM 适配器返回不一致
- 状态机：`idle / pending / streaming / completed / aborted / failed`
- `finish_reason === 'max_tokens' | 'length'` 自动显示"输出截断"警告徽章

### 2.2 输出长度自适应

- 设置项：默认 maxTokens、按输入长度自动扩容、输出膨胀因子、默认温度
- 估算公式：`charCount × 膨胀因子 + 段落格式预留(clamp 512~4096)`
- 与模型自身 `tokenLimits.output` 做 `min()` 并 clamp 到 `[256, 131072]`
- 卡片底部展示「实际 max / 模型上限」对照

### 2.3 持久化（已迁到 ConfigManager）

落盘位置：`appDataDir/translator/`

| 文件            | 内容                                                          | 防抖  |
| --------------- | ------------------------------------------------------------- | ----- |
| `settings.json` | 全局参数（maxTokens / 温度 / 流式 / 历史开关）                | 400ms |
| `history.json`  | 最近 30 条翻译历史                                            | 600ms |
| `presets.json`  | 预设数组（含每个渠道的 profileId / modelId）+ 当前激活预设 ID | 400ms |

实现细节：

- `initialize()` 并发加载三份 + LLM Profiles
- `isLoadingPersistence` 哨兵防"加载即回写"循环
- 启动时校验 `activePresetId` 是否还存在，不存在降级到首个预设
- `clearHistory()` 立即落盘空数据，不依赖防抖

### 2.4 UI / 交互

- 预设 Tab 切换栏，最多展示当前预设的渠道数 badge
- 输入面板：源/目标语言下拉 + 一键互换、Ctrl+Enter 发送（符合项目规范）
- 渠道面板：增加/删除/换模型（最多 6 个，最少 1 个）
- 结果面板：响应式 grid 并排卡片，流式自动吸底（用户手动上滑暂停，回到底部恢复）
- 设置对话框：使用 [`BaseDialog`](../../../../components/common/BaseDialog.vue) 并遵循 `lockScroll: false` 规范
- 历史 strip：最近 8 条快速回填

### 2.5 注册与排序

- [`translator.registry.ts`](../../translator.registry.ts) 同时导出 `toolConfig` + `TranslatorRegistry` 类
- 已添加到 [`src/config/tools.ts`](../../../../config/tools.ts) `DEFAULT_TOOLS_ORDER` 的"文本处理"组首位
- 图标 `markRaw(Languages)`，符合 `.kilocode/rules/rules.md` 硬性要求

## 5. 与原计划的演进差异

| 计划文档                                                 | 实际实现                                                                         | 处理                                  |
| -------------------------------------------------------- | -------------------------------------------------------------------------------- | ------------------------------------- |
| 单个 `appConfigDir/translator/presets.json`              | 拆成 `settings.json / history.json / presets.json` 三文件                        | 三文件职责更清晰，已采用              |
| `useTranslatorCore.translateParallel()` 在 core 层做并发 | 并发逻辑下沉到 Store 的 [`translate()`](../../composables/useTranslatorStore.ts) | Store 需掌控状态机/控制器，下沉更合理 |
| `TranslationResult` 字段仅 `isStreaming + error`         | 增加 `status / startedAt / appliedMaxTokens / modelOutputLimit / finishReason`   | 更细的状态机便于 UI 反馈              |
| 计划未涉及 maxTokens 自动估算                            | 完整的 `estimateTranslationOutputTokens()` + clamp 链                            | 解决长文翻译被截断问题                |
| 计划只有 `abortAll`                                      | 增加 `abortChannel(id) / retryChannel(id)`                                       | 单渠道失败可独立救                    |
| 计划建议讨论后实现语言粘性                               | 采纳方案 B（智能粘性）：用户改过则保留，否则跟随预设默认                         | 无需额外设置项，行为更直观            |
