目录清洁工具 ​

智能清理过时的缓存、存档和临时文件，让磁盘空间"呼吸"起来。

功能概述 ​

目录清洁工具帮你精准、安全地清理磁盘空间，核心能力包括：

  • 🕵️ 按规则扫描 — 支持按文件名模式（glob）、文件年龄（天）、文件大小（MB）组合过滤，精准定位目标
  • ♻️ 安全清理到回收站 — 所有被清理的文件移至系统回收站，误删可恢复，无需担心
  • ⚡ Rust 后端驱动 — 扫描清理由原生 Rust 后端执行，处理大量文件时依然丝滑流畅
  • 📊 实时进度反馈 — 扫描和清理过程实时显示进度，支持随时取消
  • 📋 内置预设场景 — 一键应用针对 VS Code 检查点、node_modules、编译产物等常见场景的预设配置
  • 🔍 结果二次筛选 — 扫描结果出来后，还可以在结果面板中按名称、年龄、大小进一步过滤

快速上手 ​

1. 打开工具 ​

在左侧工具栏中找到 目录清洁工具（图标为 🗑️），点击进入。

2. 配置扫描参数 ​

在左侧配置面板中，依次设置：

参数说明示例
目标路径要扫描的目录，支持环境变量C:/Projects、%UserProfile%/Downloads
名称模式文件名匹配规则（glob 格式），空字符串匹配所有node_modules、*.tmp、dist|build
最小文件年龄文件多久没被修改才纳入清理30 → 30 天以上未修改
最小文件大小文件至少多大才纳入清理100 → 100 MB 以上
最大扫描深度递归扫描的最大子目录层级3 → 最多往下翻 3 层

💡 提示: 参数留空表示"不限制"。比如不填"最小文件年龄"和"最小文件大小"，只填名称模式 *.log，就会扫描出目标路径下所有 .log 文件。

3. 开始扫描 ​

点击 "开始扫描" 按钮，工具会启动 Rust 后端遍历目录。右侧结果面板会实时显示扫描进度。

4. 预览结果，勾选目标 ​

扫描完成后，结果面板会列出匹配的文件和目录，包含名称、大小、修改时间等信息。

  • 勾选需要清理的项目（全选 / 反选）
  • 利用结果面板上方的二次筛选条件进一步缩小范围（如只看 100MB 以上的文件）
  • 面板顶部会显示"已选 N 项，共 X MB"，帮你确认

5. 执行清理 ​

点击 "清理所选"，系统会弹出确认对话框，显示要删除的数量和总大小。

确认后，文件将被移动到回收站。

功能详解 ​

预设配置 ​

内置的预设规则覆盖了常见清理场景，点击配置面板顶部的预设按钮即可一键填入参数：

预设名称针对场景默认参数
Kilo Code 检查点Kilo Code 任务的 .git 检查点缓存路径: AppData 下的 tasks，名称: checkpoints，7 天以上
Roo Cline 检查点Roo Cline 任务的 .git 检查点缓存同上
VS Code 扩展缓存扩展产生的缓存文件名称: *.cache，30 天以上，>= 10MB
Node.js 项目依赖长期不用的 node_modules名称: node_modules，30 天以上，>= 100MB
编译产物dist / build / out / target 目录名称: dist|build|out|target，14 天以上，>= 10MB
临时文件夹系统临时文件全部文件，7 天以上，>= 1MB

二次筛选 ​

扫描结果出来之后，结果面板顶部提供独立的过滤条件（名称、最小年龄、最小大小）。与扫描参数不同，这些过滤条件只作用于已有结果，不需要重新扫描。

  • 场景示例: 先扫描整个项目目录，然后在结果中只看 node_modules 文件夹、且大于 500MB 的项目
  • 筛选条件激活时，面板顶部会显示"已激活"标识，方便取消

停止扫描 / 清理 ​

无论是扫描还是清理，都可以随时点击 "停止" 按钮终止操作。后端会响应取消指令并立即停止。

环境变量支持 ​

目标路径支持以下环境变量，方便指向系统特殊目录：

变量示例解析结果
%AppData%C:/Users/你的用户名/AppData/Roaming
%LocalAppData%C:/Users/你的用户名/AppData/Local
%UserProfile% / $HOMEC:/Users/你的用户名

高级用法 ​

批量清理多个场景 ​

可以分多次扫描同一个目录，每次使用不同的过滤规则，清理不同的目标。

示例: 清理 D:/Projects 目录下的过期依赖和日志

第 1 次扫描: 名称模式 = node_modules，最小年龄 = 30 天，最小大小 = 50MB
→ 勾选上大项目，清理 → 释放 2.3GB

第 2 次扫描: 名称模式 = *.log，最小年龄 = 7 天
→ 全选 → 清理 → 释放 120MB

配合二次筛选精细化操作 ​

扫描结果较多时，先用二次筛选缩小范围，再勾选清理。

示例: 全盘扫描 .tmp 文件

  1. 目标路径填 C:/，名称模式填 *.tmp，最小大小填 1（只看 1MB 以上的临时文件）
  2. 扫描完成后，在二次筛选区输入 cache，只看名称或路径中包含 cache 的项目
  3. 确认无误后清理

使用环境变量预填路径 ​

快速指向 VS Code 的全局存储路径来清理编辑器缓存：

路径: %AppData%/Code/User/globalStorage
名称模式: *.cache
最小年龄: 30

常见问题 ​

清理后文件去哪里了？ ​

所有被清理的文件都会被移动到系统回收站。如果不小心误删了，可以打开回收站找到对应文件并还原。

扫描结果里的目录为什么没有大小？ ​

目录的大小是在扫描过程中实时计算的。对于包含大量子文件的目录，大小会在处理完毕后显示。如果目录正在被扫描或计算，会显示 --。

为什么扫描结果和实际文件数不符？ ​

扫描受 maxDepth（最大深度）参数限制。如果设置的最大深度为 3，工具只会扫描到第 3 层子目录。需要扫描更深的目录，适当增加该参数即可。

预设中的 %AppData% 路径在我的电脑上有效吗？ ​

有效。工具会自动将 %AppData% 等环境变量解析为当前用户的实际路径。如果你在使用 macOS 或 Linux，也可以使用 $HOME 格式的环境变量。

扫描太慢了，能取消吗？ ​

可以。扫描过程中点击 "停止" 按钮即可随时取消。取消后，已扫描到的结果会保留在结果面板中，你可以直接基于这些结果执行清理。
