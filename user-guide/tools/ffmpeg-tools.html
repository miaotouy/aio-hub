FFmpeg 多媒体工作台 ​

一站式音视频处理工具箱，基于 FFmpeg 引擎的强大能力，提供直观的图形化操作界面。

FFmpeg 多媒体工作台是 AIO Hub 内置的专业音视频处理工具。它通过图形化界面将 FFmpeg 命令行工具的复杂参数转化为直观的控件，让视频压缩、格式转换、音频提取等常见任务变得简单高效。

功能亮点 ​

  • 🎯 场景化预设：内置微信、抖音、B站、极致压缩等 8 套场景预设，一键应用即可开始处理
  • 📊 实时进度监控：毫秒级进度更新，实时显示处理速度（Speed）、码率（Bitrate）和当前时间点
  • 🔄 多任务并行：支持同时运行多个 FFmpeg 进程，任务监控面板清晰管理所有处理队列
  • 🎬 内置媒体预览：拖入文件后自动解析元数据，支持视频/音频播放预览
  • ⚡ 智能参数生成：根据你的参数配置实时生成 FFmpeg 命令行，学习 FFmpeg 的同时确保参数正确

快速上手 ​

前置准备 ​

该工具默认使用系统中注册的 ffmpeg 命令。如果尚未安装 FFmpeg，请在服务配置中设置 FFmpeg 二进制文件的实际路径。

  1. 在主页中点击「FFmpeg 工具」卡片进入工作台
  2. 点击「服务配置」标签页
  3. 填写 FFmpeg 和 FFprobe 的路径（如 C:\ffmpeg\bin\ffmpeg.exe），或确保它们已加入系统 PATH

第一次压缩视频 ​

  1. 切换到「工作台」标签页
  2. 将视频文件拖入右侧的待处理文件区域，或点击「选择文件」按钮打开系统对话框
  3. 在「处理配置」区域，从预设下拉列表中选择「微信视频号优化」
  4. 点击底部醒目的「开始处理任务」按钮
  5. 切换至「任务监控」标签页，查看实时进度

功能详解 ​

工作台主界面 ​

工作台采用左右分栏布局：

  • 左侧 — 处理配置：包含预设选择、参数表单和 FFmpeg 指令预览
  • 右侧 — 文件管理：文件拖入区、媒体元数据显示和播放预览

处理模式 ​

模式说明典型用途
视频重编码指定视频编码器（libx264/libx265）和 CRF 或码率参数压缩视频体积、调整画质
音频提取移除视频轨道，仅保留音频并转码从视频中提取背景音乐
格式转换使用 -c copy 快速转换封装格式MP4 转 MKV，无需重编码
自定义模式直接输入原始 FFmpeg 命令行参数高级用户自由发挥

参数调优指南 ​

视频编码器

  • libx264：兼容性最佳，几乎所有播放器和平台都支持
  • libx265（HEVC）：同等画质下体积小 30-50%，但兼容性略差
  • h264_nvenc：NVIDIA 显卡硬件编码，处理速度极快但画质稍逊

CRF（恒定质量因子）

  • 18-22：高画质（适合存档、B站投稿）
  • 23-25：平衡（日常使用）
  • 26-30：高压缩（适合网传分享）

任务监控 ​

「任务监控」标签页提供了完整的任务管理能力：

  • 统计概览：顶部四个统计卡片分别展示总任务数、处理中、已完成和失败数量
  • 任务表格：显示每个任务的名称、状态、处理速度/码率/时间、操作按钮
  • 操作按钮：
    • 🔴 停止按钮：终止正在处理的任务
    • ℹ️ 媒体详情：查看输出文件的完整媒体信息
    • 📂 定位文件：在文件管理器中打开输出文件所在目录
    • ➡️ 发送到：将输出文件直接发送给 LLM Chat 或转写工具
    • 🗑️ 删除记录：从任务列表中移除该记录

预设管理 ​

系统内置 8 套预设，覆盖常见场景：

预设名称核心参数适用场景
微信视频号优化libx264 + 1080p + CRF 23微信朋友圈/视频号
抖音/快手上传libx264 + 码率 4000k短视频平台
极致压缩 (H.265)libx265 + CRF 28 + 720p存档备份，最小体积
高保真音频提取AAC 320kbps提取高质量伴奏
无损音频提取FLAC 无损音频存档
快速转换 (流拷贝)-c copy 不重编码仅转换封装格式
B站投稿优化libx264 + CRF 20 + slow满足 B站 二压标准
4K HDR 保留libx265 + CRF 22 + 10bit高画质素材归档

你也可以将当前参数保存为自定义预设，支持重命名、删除、导出为 JSON 文件分享给他人，或导入他人分享的预设。

打开「专业模式」开关后，参数表单会显示更多高级选项，如像素格式（pixel format）、音频声道数等。

服务配置 ​

在「服务配置」标签页可以设置：

  • FFmpeg 路径：FFmpeg 可执行文件的路径
  • FFprobe 路径：FFprobe 可执行文件的路径（用于详细媒体信息分析）
  • 默认输出目录：处理完成后文件的存放位置
  • 最大并发任务数：同时处理的 FFmpeg 进程数量，默认为 2

高级用法 ​

跨工具协作 ​

完成任务后，在「任务监控」表格中点击已完成任务的「发送到」按钮，可以：

  • 发送到 LLM Chat：将输出的音频/视频文件直接作为 LLM 对话的附件
  • 发送到转写工具：将输出的音频文件直接发送给语音转文字工具

预设导出与分享 ​

  1. 在工作台左侧的预设选择区域点击「管理预设」（齿轮图标）
  2. 在管理弹窗中，点击任意预设（内置或自定义）右侧的「导出」按钮
  3. 系统会下载一个 JSON 文件，其中包含该预设的完整参数快照
  4. 将 JSON 文件分享给其他人，对方点击管理弹窗底部的「导入预设」即可使用

配合专业模式使用 ​

开启「专业模式」后，参数表单会显示更多高级选项：

  • 像素格式（pixel format）：如 yuv420p（通用）、yuv422p（高质量）、yuv420p10le（10bit HDR）
  • 音频采样率：如 44100（CD 音质）、48000（专业音质）
  • 音频声道数：单声道（1）或立体声（2）
  • 音频编码器：支持 libmp3lame（MP3）、libopus（Opus）、copy（流拷贝）等

常见问题 ​

Q：为什么处理失败？ ​

请按以下顺序排查：

  1. 确认 FFmpeg 路径配置正确（可以在系统终端中手动运行 ffmpeg -version 测试）
  2. 查看「实时控制台」中的 FFmpeg 日志输出，具体错误信息会指导你定位问题
  3. 检查输入文件是否损坏或被其他程序占用
  4. 确认输出目录有写入权限且磁盘空间充足

Q：如何实现最快速度的格式转换？ ​

选择「快速转换 (流拷贝)」预设。该模式使用 -c copy 参数，不重新编码音视频流，仅转换封装格式（如 MKV → MP4），速度几乎与文件 I/O 一致。

Q：如何得到最小的文件体积？ ​

选择「极致压缩 (H.265)」预设，或手动调高 CRF 值（如 30-35）并降低分辨率。注意：过度压缩会导致画质严重下降，建议先在部分片段上测试效果。

Q：为什么系统没有检测到 FFmpeg？ ​

  • Windows：检查系统环境变量 PATH 是否包含 FFmpeg 的 bin 目录。重启应用后生效。
  • macOS/Linux：使用包管理器安装后通常自动注册。也可通过 which ffmpeg 确认安装路径，然后在服务配置中手动填写。

Q：任务监控中的"应用重启，任务已自动取消"是什么意思？ ​

FFmpeg 进程是独立于应用运行的操作系统进程。如果 AIO Hub 意外关闭，正在执行的 FFmpeg 任务会随之终止。应用重启后，会自动将这些未完成的任务标记为「已取消」状态，避免状态不一致。你可以清除这些历史记录后重新提交任务。
