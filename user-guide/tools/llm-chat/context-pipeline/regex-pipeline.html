正则管道 (Regex Pipeline) ​

正则管道是 LLM Chat 中功能最强大的文本处理系统，它允许你通过正则表达式和自定义脚本对消息内容进行全自动的清洗、转换和增强。

核心概念 ​

正则管道基于“匹配-替换”逻辑，但针对聊天场景进行了深度定制：

  • 多层级合并：规则可以定义在全局、智能体或用户层级，系统会自动合并应用。
  • 双阶段应用：区分“渲染层”（仅影响你看到的 UI）和“请求层”（影响发送给 AI 的上下文）。
  • 脚本引擎：除了简单的文本替换，还支持使用 JavaScript 编写复杂的处理逻辑。

三层配置体系 ​

系统从三个来源收集正则规则，并按优先级合并：

  1. 全局配置 (Global)：在“设置 -> 插件/正则”中定义，适用于所有会话。
  2. 智能体配置 (Agent)：在智能体编辑器中定义，仅当该智能体参与对话时生效。
  3. 用户配置 (User)：在用户个人资料中定义，仅当该用户发送消息时生效。

绑定模式 (Binding Mode) ​

在正则配置面板中，你可以选择规则的绑定策略：

  • 跟随消息 (Message Mode)：默认模式。每条消息在生成时会“冻结”当前的正则配置。即使后续修改了智能体的正则规则，旧消息的渲染也不会改变。
  • 跟随会话 (Session Mode)：所有消息始终使用当前会话中智能体/用户的最新正则配置进行渲染。

双管道集成 ​

规则可以根据其用途，被指派到不同的处理阶段：

1. 渲染管道 (Render Pipeline) ​

  • 触发时机：消息显示在屏幕上之前。
  • 用途：UI 增强（如将 <think> 标签转为折叠框）、隐私脱敏（隐藏 API Key）、口癖转换、格式美化。
  • 特性：不改变数据库中的原始存储，也不影响发送给 AI 的内容。

2. 请求管道 (Request Pipeline) ​

  • 触发时机：消息发送给 LLM 接口之前。
  • 用途：上下文压缩（移除不必要的冗余信息）、提示词注入、特定格式强制转换。
  • 特性：直接影响 AI 接收到的文本，是实现高级 Agent 逻辑的关键。
