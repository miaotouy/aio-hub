系统脉搏 ​

概述 ​

系统脉搏是一个实时硬件监控仪表盘，让你像看心电图一样随时掌握电脑的运行状态。它覆盖 CPU、内存、磁盘、网络和 GPU 五个维度的关键指标，并配有历史趋势折线图，帮助你快速发现性能瓶颈和异常。

核心功能 ​

  • 五大维度实时监控：CPU（含多核热力图）、内存（含 Swap）、磁盘（容量 + I/O 速率）、网络（上/下行）、GPU（含 NVIDIA 编解码）
  • 历史趋势折线图：每个维度都配有 60 秒滚动趋势图，变化一目了然
  • GPU 深度分析：支持 NVIDIA 显卡的编解码器使用率监控，适合视频创作者和 AI 开发者
  • 三种 UI 密度：紧凑 / 标准 / 宽大三档布局，适应不同屏幕尺寸和信息密度偏好
  • 状态快照导出：一键复制当前状态为文本，或导出完整历史记录为 JSON 文件

快速上手 ​

三步开始监控 ​

第一步：打开工具

在左侧工具列表中找到 系统脉搏，点击进入。你会看到页面显示"监控已暂停"的空状态，中间有一个 立即开启 按钮。

  2. 监控卡片网格 — 每个维度一张毛玻璃卡片，包含核心数值、历史折线图和详细指标

功能详解 ​

状态栏（顶部摘要条） ​

位于页面最上方，用紧凑的一行展示所有关键维度的实时概览：

指标颜色标识说明
CPU按使用率变色全局 CPU 使用率百分比
内存正常物理内存占用百分比
↑ 网络上行蓝色所有网卡上传速率之和
↓ 网络下行橙色所有网卡下载速率之和
GPU0 / GPU1...按使用率变色每张显卡的使用率百分比

右侧还有一个实时更新的时钟，方便你截图记录时参考时间戳。

CPU 监控卡片 ​

CPU 卡片是信息最丰富的区域，包含：

  • 使用率折线图：过去 60 秒的全局 CPU 使用率变化曲线
  • 实时频率：当前 CPU 实际运行频率（Windows 下通过 PDH 计数器获取，含睿频信息）
  • 基准频率：CPU 的基础标称频率
  • 温度：CPU 当前温度（平台支持时显示，按温度变色：<50° 正常、50-80° 橙色、>80° 红色）
  • 进程数：当前正在运行的进程数量
  • 运行时间：系统已连续运行的总时长
  • 多核热力矩阵：每个逻辑核心以一个小色块表示，颜色越深代表该核心负载越高（蓝→橙→红渐变），一眼看出哪些核心在"摸鱼"、哪些在"拼命"

内存监控卡片 ​

  • 使用率折线图：过去 60 秒的内存占用趋势
  • 物理内存（RAM）条形图：已用 / 总量，附带百分比
  • Swap 条形图：如果系统启用了 Swap 且总量大于 0 会显示，使用橙色渐变条

磁盘监控卡片 ​

磁盘卡片采用列表形式展示所有挂载点，每个磁盘包含：

  • 磁盘名称和挂载点：如 C:、D: 等
  • 容量条形图：已用 / 总量，百分比数字
  • I/O 读写速率：当前读取速率（绿色）和写入速率（橙色）
  • I/O 趋势迷你图：双系列折线图（绿色读取 + 橙色写入），展示 I/O 变化趋势

网络监控卡片 ​

  • 上/下行折线图：两条独立的折线图，上行蓝色、下行橙色
  • 总速率显示：卡片顶部的 ↑ ↓ 数值为所有网卡的总和
  • 接口详情列表：列出每个网卡独立的速率，列表最多显示 120px 高度，超过可滚动查看

GPU 监控卡片 ​

每张 NVIDIA 独立显卡会生成一张独立卡片，支持多卡环境（如双卡工作站）：

  • 使用率折线图：过去 60 秒的 GPU 使用率
  • 温度：显卡当前温度，颜色随温度变化
  • 显存条形图：已用 / 总量，蓝色渐变条
  • 编码器使用率（NVIDIA 专属）：视频编码引擎的负载百分比
  • 解码器使用率（NVIDIA 专属）：视频解码引擎的负载百分比

工具栏操作 ​

页面右上角的工具栏提供了三种操作：

操作图标说明
复制状态📋将当前快照格式化为易读的文本报告，复制到剪贴板，方便粘贴到聊天工具或 Issue 中
导出历史⬇️弹出系统保存对话框，将最近 60 个采样点的完整快照导出为 JSON 文件
切换密度▦在紧凑（Small）/ 标准（Medium）/ 宽大（Large）三档之间切换，持久化到本地存储

三态视图 ​

工具会自动根据当前监控状态展示不同的界面：

状态展示内容
从未启动空状态提示 + 蓝色"立即开启"按钮
启动中旋转加载图标 + "正在采集系统数据…"文字
正常运行或暂停后完整的仪表盘（暂停后保留最后一帧数据，不会闪烁或消失）

主题适配 ​

所有监控卡片都遵循项目的主题外观系统，支持毛玻璃效果和透明度调整。你可以在设置中调整界面质感，监控卡片会自动适配。

高级用法 ​

监控暂停后分析数据 ​

点击右上角的开关暂停监控后，仪表盘会保留最后一帧数据，不会清空。你可以在这个"冻结"的状态下仔细分析各项指标，或者点击复制按钮将当前数据分享给他人。

使用紧凑模式腾出屏幕空间 ​

如果你需要一边监控一边做其他工作，或者屏幕尺寸较小，可以通过工具栏左侧的布局按钮切换到 紧凑（Small） 模式。紧凑模式下卡片间距更小、字体更紧凑、折线图更矮，能在有限空间内展示更多信息。

导出 JSON 历史进行深度分析 ​

工具保留最近 60 秒的完整快照历史。点击导出按钮，会弹出保存对话框让你选择存储位置。导出的 JSON 文件包含每个时间点的完整系统状态（CPU、内存、磁盘、网络、GPU），你可以用其他工具（如 Excel、Python）做进一步分析，比如：

  • 绘制长时间段的性能趋势图
  • 分析特定进程（进程数指标）与 CPU 负载的关系
  • 对比不同时间段的内存使用峰值

配合其他工具使用 ​

当你使用 视频转码工具 或 AI 对话 等计算密集型工具时，可以同时打开系统脉搏观察资源消耗情况——比如查看 GPU 使用率是否跑满、内存是否接近上限、CPU 温度是否过高。

常见问题 ​

Q：为什么我看不到 GPU 卡片？ ​

系统脉搏使用 NVIDIA Management Library（NVML）来获取 GPU 数据，因此仅支持 NVIDIA 显卡。如果你的电脑使用的是 AMD 或 Intel 集成显卡，GPU 区域将不会显示任何卡片。这是已知限制，不影响其他指标的监控。

Q：CPU 温度显示为"—"或空白？ ​

CPU 温度的读取依赖于 sysinfo 库的 Components 接口，该接口在部分 Windows 系统上无法正确读取温度传感器数据。这种情况下温度字段会返回 null，卡片上该条目不会显示。这不是硬件故障，只是数据不可获取。

Q：磁盘的 I/O 读写速率不准确？ ​

在 Windows 系统上，工具通过 PDH 计数器获取磁盘 I/O 速率，但 PDH 提供的是 全盘汇总值。目前实现将汇总值填入系统盘（如 C:\），其余磁盘的 I/O 字段显示为 0。这意味着多磁盘环境下只有系统盘的 I/O 数据是准确的。这是当前版本的已知限制。

Q：暂停后数据会不会丢失？ ​

不会。暂停监控时，系统会保留最后一帧快照和所有历史数据。你可以继续查看、复制或导出已采集的数据。只有关闭工具页面或刷新后数据才会被清除。

Q：历史趋势图能看更长时间的记录吗？ ​

目前每个维度的历史趋势图固定保留最近 60 个采样点（约 60 秒）。这是一个定长环形缓冲区（RingBuffer）设计，目的是平衡内存开销和可视化效果。如果你需要更长时间的记录，可以使用导出功能将完整快照保存为 JSON 文件，再用其他工具分析。
