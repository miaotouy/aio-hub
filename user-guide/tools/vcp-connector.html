VCP 连接器 ​

概述 ​

VCP 连接器是 AIO Hub 与 VCP (Variable & Command Protocol) 生态系统之间的桥梁。它让你能在 AIO Hub 中实时观察 VCP 的运行状态，又能将 AIO Hub 的工具能力共享给 VCP 网络中的 AI Agent，实现双向能力互通。

简单来说，VCP 连接器就是你的 VCP 可视化控制台——既能看，又能用。

核心功能 ​

  • 实时消息监控：以可视化的卡片形式实时接收 VCP 服务器的运行时广播消息，包括 RAG 检索详情、元思考链推理过程、Agent 私聊预览、AI 记忆回溯和插件执行状态，是调试 VCP 工作流的绝佳窗口
  • 分布式节点：将 AIO Hub 注册为 VCP 分布式网络中的一个节点，让你的 AI Agent 能远程调用 AIO Hub 的本地工具（如文件操作、数据处理等）
  • 工具桥接（双向共享）：动态发现 VCP 服务器上的原生插件，并将其映射为 AIO Hub 的内部工具；反过来也能将 AIO Hub 的工具暴露给 VCP 网络。一套连接，双重共享
  • 连接配置自动探测：只需指定 VCP 安装目录，系统自动读取 config.env 文件，解析出 WebSocket 地址和鉴权 Key，无需手动记忆配置
  • 分离窗口支持：消息监控面板可以拖拽为独立窗口，让你在工作的同时实时观察 VCP 运行状态

快速上手 ​

准备工作 ​

在开始之前，你需要有一个正在运行的 VCP 服务器。VCP 连接器是一个客户端工具，它通过 WebSocket 与 VCP 服务器建立连接。

首次连接 4 步走 ​

第一步：打开 VCP 连接器

在左侧工具列表中找到 VCP 连接器，点击进入。你会看到左右分栏的布局：左侧是配置面板，右侧是消息监控和分布式节点两个标签页。

TODO: 截图 - VCP 连接器整体界面，标注左侧配置栏和右侧 Tab 标签

第二步：配置连接

在左侧的"VCP 连接与过滤"面板中，你可以选择以下两种方式之一来配置连接：

  • 方式一（推荐）：选择 VCP 安装目录 —— 点击最上方的文件夹图标按钮，从文件系统选择你的 VCP 安装目录。系统会自动读取该目录下的 config.env 文件，自动填入 WebSocket 地址和 VCP Key。
  • 方式二：手动填写 —— 如果 VCP 服务器不在本地或无法自动探测，可以手动输入 WebSocket 地址和 VCP Key。

填写完成后，开启 自动连接 开关（可选，开启后应用启动时自动连接），然后点击 连接 按钮。

第三步：确认连接状态

连接成功后，左侧面板会显示连接状态（绿色标识），同时右侧消息监控区域会开始接收来自 VCP 服务器的广播消息。

第四步：开始使用

连接建立后，你就可以：

  • 切换到 消息监控 Tab 查看 VCP 的实时运行日志
  • 切换到 分布式节点 Tab 管理暴露给 VCP 的工具
  • 如果需要，开启 工具桥接 来使用 VCP 服务器上的插件

消息监控 ​

消息监控标签页是 VCP 运行状态的"可视化仪表盘"，以卡片流的形式实时展示 VCP 广播的所有运行时事件。

消息流 ​

消息按倒序排列（最新消息在顶部），支持虚拟滚动，即使有数千条消息也能保持流畅。每条消息都以卡片形式呈现，包含：

  • 类型图标与颜色：根据消息类型自动匹配对应的图标和颜色标识
  • 时间戳：消息的精确时间
  • 内容详情：不同类型展示不同的核心信息

分布式节点 ​

分布式节点标签页让你能将 AIO Hub 注册为 VCP 网络中的一个"能力节点"，VCP 中的 AI Agent 可以通过网络远程调用你暴露的工具方法。

暴露工具管理 ​

在分布式节点模式下，你可以精细控制哪些工具暴露给 VCP 网络：

  • 自动发现开关：开启后自动扫描所有标记为 agentCallable 或 distributedExposed 的工具方法
  • 手动添加：从所有已注册的工具中搜索并添加特定工具到暴露列表
  • 工具分类标识：
    • 🏷️ 内置：VCP 协议强制要求的工具（如文件读取），不可禁用或移除
    • 🔄 自动：通过自动发现扫描到的 AI 工具，可启用/禁用
    • ✋ 手动：你手动添加到暴露列表的工具，可启用/禁用/移除
  • 同步状态：每个工具右侧显示其是否已成功同步到 VCP 服务器

TODO: 截图 - 暴露工具列表，标注内置/自动/手动的分类标识

安全提示：只有明确标记为 distributedExposed: true 的工具方法才能被远程调用。这意味着你可以精细控制哪些能力对外开放，避免敏感操作被未授权调用。

工具桥接 ​

工具桥接是 VCP 连接器的"双向能力共享"功能。开启后，VCP 服务器上的原生插件会被动态发现并映射为 AIO Hub 的内部工具——这样一来，AIO Hub 的本地 Agent 也能使用 VCP 的插件能力。

开启桥接 ​

  1. 在 分布式节点 Tab 中找到"桥接工具"区域
  2. 打开全局开关启用桥接功能
  3. 系统会自动从 VCP 服务器拉取可用插件清单

TODO: 截图 - 桥接工具列表，标注全局开关

权限控制 ​

桥接工具支持在命令级别进行精细的权限管理：

  • 启用/禁用：你可以单独启用或禁用某个远程插件，防止 Agent 调用不安全的远程命令
  • 配置同步：VCP 插件的配置项会同步到 AIO Hub，支持在 AIO UI 中直接修改远程插件的参数

表情包集成 ​

如果 VCP 服务器配置了图片服务（Image_Key），VCP 连接器会自动加载 VCP 的表情包库，让 AIO Hub 的聊天功能可以使用 VCP 服务器上的表情资源。

高级用法 ​

自动启动与后台连接 ​

在左侧配置面板中开启 自动连接 开关后，每次启动 AIO Hub 时都会自动尝试连接 VCP 服务器。配合 自动注册工具 开关，VCP 连接器会在后台静默完成连接和工具注册——即使你没有打开 VCP 连接器的界面。

这对于设置了"跟随应用自启"的 VCP 服务器特别有用——两者启动后自动建立连接，无需任何手动操作。

同时使用三条连接 ​

VCP 连接器实际上会管理多条 WebSocket 连接，每条都有独立的连接守卫和心跳机制：

  • Observer 连接：接收 VCP 广播的运行时消息（监控功能）
  • Distributed 连接：处理工具注册、远程调用和心跳上报（分布式节点功能）
  • 桥接通道：通过 Distributed 连接进行插件清单拉取和远程代理调用

三条连接相互独立，一条连接出现问题时不会影响其他连接的正常运作。

结合分离窗口构建监控屏 ​

如果你有多显示器环境，可以这样操作：

  1. 打开 VCP 连接器，建立连接
  2. 点击消息监控面板的 分离 按钮，将监控窗口拖拽到副显示器
  3. 在主显示器上继续使用其他 AIO Hub 工具
  4. 副显示器上实时展示 VCP 的运行状态

这是一种非常高效的"一边干活、一边监控"的工作模式。

使用通知系统跟进 VCP 任务 ​

VCP 运行日志中的特定事件会通过 AIO Hub 的通知系统推送到你：

  • 错误事件：VCP 工具执行报错时，自动推送错误通知，可点击查看详情
  • 任务启动：VCP 任务创建时，推送任务启动通知
  • 完成提示：任务归档、完成或成功时，以浮动提示形式展示

你可以在 AIO Hub 的 通知中心 中查看所有 VCP 相关的历史通知。

常见问题 ​

Q：连接不上 VCP 服务器怎么办？ ​

最常见的原因是 VCP 服务器没有运行。请先确认 VCP 服务器已启动并正常运行。

如果服务器确认在运行但仍无法连接，可以尝试：

  1. 检查 WebSocket 地址格式是否正确（标准格式：ws://IP:端口）
  2. 确认 VCP Key 是否正确（可以在 VCP 安装目录下的 config.env 文件中找到）
  3. 检查防火墙是否阻止了连接
  4. 如果是远程服务器，确保服务器允许 WebSocket 连接

Q：消息监控面板没有显示任何消息？ ​

可能的原因：

  1. 未建立连接：检查连接状态是否为"已连接"，如果不是请点击连接按钮
  2. 连接模式不对：如果选择了 distributed 模式，Observer 连接不会建立，自然收不到广播消息。请切换为 both 或 observer 模式
  3. 暂停状态：检查消息过滤面板的"暂停"按钮是否被激活，再次点击即可恢复
  4. 类型过滤：检查类型过滤按钮是否全部被选中（可以点击"全部"按钮重置）
  5. VCP 服务器没有触发事件：VCP 只在有实际运行时事件时才会广播消息，这是正常现象

Q：暴露给 VCP 的工具太多，怎么控制？ ​

你可以在 分布式节点 > 暴露工具列表 中进行精细控制：

  1. 关闭 自动注册工具 开关，这样只有你手动添加的工具才会被暴露
  2. 在工具列表中，点击工具右侧的开关来临时禁用某个工具
  3. 对于手动添加的工具，可以直接从列表中移除

这样你就可以只暴露那些你确实想让 VCP Agent 调用的工具了。

Q：工具桥接和分布式节点有什么区别？ ​

这是两个互补但不同的功能：

  • 分布式节点：AIO Hub 作为节点加入 VCP 网络，AIO 的工具可以被 VCP 中的 Agent 调用（单向：AIO → VCP）
  • 工具桥接：VCP 服务器上的插件被映射到 AIO Hub 内部，AIO 的本地 Agent 也可以调用 VCP 的插件（反向：VCP → AIO）

简单来说，分布式节点是"给出去"，工具桥接是"拿回来"——两者结合实现双向工具共享。

Q：断开连接后重新连接需要手动操作吗？ ​

不需要。如果你开启了 自动连接 开关：

  • 网络断开后，系统会自动尝试重连
  • 重连策略采用指数退避（1 秒 → 2 秒 → 4 秒 → ... → 最多 30 秒），外加随机抖动，避免多个连接同时重连造成服务器压力
  • 连接恢复后会自动继续接收消息和注册工具

如果你没有开启自动连接，则需要手动点击 连接 按钮。

TIP

如果你需要快速重置连接，可以直接点击 断开 后再点 连接。
